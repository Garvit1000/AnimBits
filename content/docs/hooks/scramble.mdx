## Overview

The `useScramble` hook creates a "decoding" text animation where random characters scramble before revealing the final text. Perfect for tech-themed interfaces, loading states, or dramatic text reveals.

<Callout type="info">
  **Pure React:** No motion library required - uses React state and intervals for the animation.
</Callout>

    ```tsx title="lib/hooks/use-scramble.ts"
import * as React from "react";

export interface UseScrambleOptions {
  text?: string;
  duration?: number;
  delay?: number;
  scrambleChars?: string;
}

export function useScramble(options: UseScrambleOptions): string {
  const {
    text = "",
    duration = 0.8,
    delay = 0,
    scrambleChars = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789!@#$%^&*()",
  } = options;

  const [displayText, setDisplayText] = React.useState(text);

  React.useEffect(() => {
    const iterations = Math.floor(duration * 60); // 60fps
    let currentIteration = 0;

    const timeout = setTimeout(() => {
      const interval = setInterval(
        () => {
          if (currentIteration >= iterations) {
            setDisplayText(text);
            clearInterval(interval);
            return;
          }

          const progress = currentIteration / iterations;
          const newText = text
            .split("")
            .map((char, index) => {
              if (char === " ") return " ";
              if (index / text.length < progress) {
                return char;
              }
              return scrambleChars[
                Math.floor(Math.random() * scrambleChars.length)
              ];
            })
            .join("");

          setDisplayText(newText);
          currentIteration++;
        },
        (duration * 1000) / iterations,
      );

      return () => clearInterval(interval);
    }, delay * 1000);

    return () => clearTimeout(timeout);
  }, [text, duration, delay, scrambleChars]);

  return displayText;
}
```

## Use cases

`useScramble` returns a `string` (the current animated text). It does not return `MotionProps`; there is no `motion.*` element involved. Instead, render the returned string inside any element. The hook runs a `setInterval` internally, firing approximately 60 times per second for `duration * 60` iterations, replacing un-decoded characters with random ones from `scrambleChars`.

- **Hero headlines on a tech or developer-focused site.** A heading like "Build faster" that scrambles before resolving on page load establishes a code-like aesthetic immediately.
- **Loading state labels in terminal-style UIs.** Showing `"CONNECTING..."` scrambled while a WebSocket connects, then resolving to `"CONNECTED"`, communicates progress without a traditional spinner.
- **Hover-activated text reveals on portfolio items.** Trigger a new `text` value on hover to re-run the scramble effect over a project title or role label.
- **Staggered intro sequences.** Call the hook three times with increasing `delay` values to reveal a tagline word by word in a terminal boot sequence style.
- **Interactive command palette labels.** A label that scrambles briefly before showing the matched command adds a sci-fi character to a keyboard-first interface.

## When to use / when not to use

**Use `useScramble` when:**

- You want a pure-React text animation with no `motion.*` element required.
- The aesthetic is explicitly technical, terminal, cyberpunk, or matrix-themed.
- The text is short (under 40 characters). Longer text scrambles slowly and can look noisy.

**Avoid `useScramble` when:**

- The content is user-generated text that the user typed themselves. Scrambling their own input would be confusing.
- The text is inside a form or an interactive input. Screen readers re-read text nodes when their content changes, so a 60fps interval firing `setDisplayText` will cause screen readers to read dozens of random characters.
- The text changes frequently based on external data. Each change of the `text` option restarts the animation. Rapid re-renders with changing `text` props can cause the animation to never fully resolve.

## Accessibility notes

`useScramble` updates a string state at ~60fps for the duration of the animation. During scrambling, the displayed string is meaningless to screen readers.

- Wrap the animated element in an `aria-label` or `aria-live="off"` to suppress screen reader announcements during the scramble, and use `aria-label` to provide the final text upfront:

```tsx
"use client"
import { useScramble } from "@/lib/use-scramble"

export function AccessibleScramble({ finalText }: { finalText: string }) {
  const displayText = useScramble({ text: finalText, duration: 0.8 })

  return (
    <h1 aria-label={finalText} className="font-mono text-4xl font-bold">
      {displayText}
    </h1>
  )
}
```

- Respect `prefers-reduced-motion` by returning the final text immediately instead of scrambling it:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { useScramble } from "@/lib/use-scramble"

export function ReducedScramble({ text }: { text: string }) {
  const reduced = useReducedMotion()
  const displayText = useScramble({ text, duration: reduced ? 0 : 0.8 })

  return (
    <span className="font-mono" aria-label={text}>
      {displayText}
    </span>
  )
}
```

Setting `duration: 0` makes `iterations = 0`, so `setDisplayText(text)` is called on the first interval tick and the scramble resolves immediately.

## Performance considerations

- `useScramble` uses `setInterval` at a rate of `(duration * 1000) / (duration * 60)` ms per tick, which equals approximately 16.7ms (60fps). This is a JavaScript-main-thread interval that calls `setDisplayText` and triggers one React state update per frame.
- Each state update schedules a React render and a DOM text node update. For a single element this is negligible, but avoid running multiple `useScramble` instances simultaneously on a page if the interval is rapid (duration under 0.5s).
- The `setTimeout` wrapper respects the `delay` option and does not start the interval until after the delay elapses. The timeout is cleaned up in the effect's return function, preventing memory leaks on unmount.
- The interval fires for exactly `Math.floor(duration * 60)` iterations and then clears itself. There is no ongoing cost after the animation completes unless `text` changes, which restarts the effect.
- Prefer monospace fonts (`font-mono`) to prevent text-width jitter during scrambling. Variable-width fonts cause the layout to shift on every character change.

### Hover-triggered scramble reveal

```tsx
"use client"
import { useState } from "react"
import { useScramble } from "@/lib/use-scramble"

export function HoverReveal({ secret }: { secret: string }) {
  const [active, setActive] = useState(false)
  const display = useScramble({ text: active ? secret : "???", duration: 0.6 })

  return (
    <button
      onMouseEnter={() => setActive(true)}
      onMouseLeave={() => setActive(false)}
      aria-label={`Reveal: ${secret}`}
      className="font-mono text-lg text-neutral-400 hover:text-neutral-900"
    >
      {display}
    </button>
  )
}
```
