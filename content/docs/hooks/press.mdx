## Overview

The `usePress` hook adds a press-down effect when clicking or tapping elements, providing tactile feedback. Perfect for buttons, cards, and any interactive element that needs visual confirmation of user interaction.

<Callout type="info">
  **Tactile Feedback:** This hook uses `whileTap` to provide immediate visual feedback, making your UI feel more responsive and interactive.
</Callout>

import { type MotionProps, type Easing } from "motion/react";

export interface UsePressOptions {
  pressScale?: number;

  duration?: number;

  ease?: Easing | Easing[];

  addRotation?: boolean;

  rotationAmount?: number;
}

export function usePress(options: UsePressOptions = {}): MotionProps {
  const {
    pressScale = 0.95,
    duration = 0.1,
    ease = "easeInOut",
    addRotation = false,
    rotationAmount = 2,
  } = options;

  return {
    whileTap: {
      scale: pressScale,
      rotate: addRotation ? rotationAmount : 0,
    },
    transition: {
      duration,
      ease,
    },
  };
}

## Composing with Other Hooks

Combine with hover effects for rich interactions:

```tsx
import { usePress } from "@/lib/hooks/use-press"
import { useHoverLift } from "@/lib/hooks/use-hover-lift"

export function RichButton() {
  const press = usePress({ pressScale: 0.92 })
  const lift = useHoverLift({ liftDistance: 6 })
  
  // Merge the animations
  const combined = {
    ...lift,
    ...press,
  }
  
  return (
    <motion.button {...combined} className="btn">
      Hover + Press
    </motion.button>
  )
}
```

## Use cases

`usePress` returns `MotionProps` with a `whileTap` object (containing `scale` and `rotate`) and a `transition` object. Spread the props onto any `motion.*` element and the element will compress when clicked or tapped, then spring back.

- **Primary and secondary action buttons.** Every click deserves a physical response. A `motion.button` with `usePress()` at the default `pressScale: 0.95` gives every submit, confirm, or cancel button a tactile click feel.
- **Icon-only toolbar buttons.** Tight icon buttons (share, like, bookmark) benefit from `pressScale: 0.9` so the scale change is noticeable on a small target.
- **Clickable cards.** A card that navigates on click can use `pressScale: 0.98` for a barely-there press that signals interactivity without making the card feel fragile.
- **Image thumbnails in a gallery.** A `motion.img` or `motion.button` wrapping a thumbnail with `usePress({ pressScale: 0.96 })` lets the user know the image responds to their tap before a lightbox opens.
- **Chip and tag filters.** Toggle chips with `usePress({ pressScale: 0.92, addRotation: true, rotationAmount: 2 })` make filter selection feel playful.

## When to use / when not to use

**Use `usePress` directly when:**

- You need the press effect on a custom element, not a prebuilt component.
- You want to compose the press with [useHoverLift](/docs/hooks/hover-lift): since both return plain `MotionProps`, you can spread them both onto the same `motion.button` and the `whileHover` and `whileTap` props do not conflict.
- You need `addRotation: true` for a playful icon button where a tiny twist on press adds personality.

**Reach for the prebuilt component when:**

- You just need a standard styled button with a press effect and no custom host element.

**Avoid `usePress` when:**

- The element is a text input, textarea, or select. Users hold these down to select text, and a scale-down during the hold will distort the text cursor position.
- The element is a drag handle. `whileTap` fires immediately on pointer down, and scaling the handle while dragging can shift the drag origin.
- The press scale would be applied to a layout container that holds many children; scaling the container changes all children's apparent positions and can cause visual glitches.

## Accessibility notes

`usePress` uses `whileTap`, which fires on `pointerdown` and ends on `pointerup` or `pointercancel`. This works for mouse clicks, touch taps, and stylus presses. It does not fire on keyboard Enter or Space.

- **Keyboard users.** Pressing Enter or Space on a `<button>` triggers the button's `click` event but does not trigger `whileTap`. The visual press animation is absent for keyboard users. This is acceptable as long as the button has a visible `:focus-visible` ring that communicates focus.
- **Respect `prefers-reduced-motion`.** A fast scale change (default `0.1s`) is unlikely to cause vestibular issues, but best practice is to honor the setting:

```tsx
"use client"
import { motion } from "motion/react"
import { useReducedMotion } from "motion/react"
import { usePress } from "@/lib/use-press"

export function ActionButton({ children, onClick }: { children: React.ReactNode; onClick: () => void }) {
  const reduced = useReducedMotion()
  const pressProps = usePress({ pressScale: 0.95 })

  return (
    <motion.button
      onClick={onClick}
      {...(reduced ? {} : pressProps)}
      className="rounded-lg bg-blue-600 px-6 py-2.5 text-sm font-semibold text-white focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-blue-500"
    >
      {children}
    </motion.button>
  )
}
```

- If using `addRotation: true`, keep `rotationAmount` small (1 to 3 degrees). Larger rotations on buttons can make them appear broken to users who are not expecting the effect.

## Performance considerations

- `usePress` uses `whileTap` and animates only `scale` (and optionally `rotate`). Both are transforms and run on the compositor with no layout or paint.
- The default `duration: 0.1s` means the animation completes in 6 frames at 60fps. The total compositor cost is minimal.
- Because the hook returns a plain `MotionProps` object, it is reconstructed on every render of the calling component. If the parent re-renders often, pass a stable options object (or call `usePress` outside of frequently-rendered components).
- For elements that receive very rapid clicks (double-click actions, spam-clicking), Framer Motion interrupts and reverses the current animation immediately when a new press starts. This is handled gracefully and does not cause visual glitches.

## Real-world examples

### Submit button that disables while loading

```tsx
"use client"
import { useState } from "react"
import { motion } from "motion/react"
import { usePress } from "@/lib/use-press"

export function SubmitButton() {
  const [loading, setLoading] = useState(false)
  const pressProps = usePress({ pressScale: 0.95 })

  async function handleSubmit() {
    setLoading(true)
    try {
      await submitForm()
    } finally {
      setLoading(false)
    }
  }

  return (
    <motion.button
      onClick={handleSubmit}
      disabled={loading}
      {...(loading ? {} : pressProps)}
      className="flex h-11 items-center justify-center gap-2 rounded-xl bg-blue-600 px-8 text-sm font-semibold text-white disabled:cursor-not-allowed disabled:opacity-60"
    >
      {loading ? "Saving..." : "Save changes"}
    </motion.button>
  )
}
```

### Keyboard shortcut chip with press feedback

```tsx
"use client"
import { motion } from "motion/react"
import { usePress } from "@/lib/use-press"

export function ShortcutChip({ keys, label, onActivate }: { keys: string; label: string; onActivate: () => void }) {
  const pressProps = usePress({ pressScale: 0.92, duration: 0.08 })

  return (
    <motion.button
      onClick={onActivate}
      {...pressProps}
      aria-label={label}
      className="inline-flex items-center gap-1.5 rounded-md border border-neutral-200 bg-neutral-50 px-2.5 py-1 text-xs font-mono text-neutral-600 dark:border-neutral-700 dark:bg-neutral-800 dark:text-neutral-400"
    >
      {keys}
    </motion.button>
  )
}
```
