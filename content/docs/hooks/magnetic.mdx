## Overview

The `useMagnetic` hook creates a magnetic attraction effect where elements smoothly follow the cursor on hover using spring physics. Perfect for creating engaging, interactive UI elements.

<Callout type="info">
  **Spring-based animation:** Uses Framer Motion's spring physics for natural, fluid movement.
</Callout>

    ```tsx title="lib/hooks/use-magnetic.ts"
import * as React from "react";
import { useMotionValue, useSpring } from "motion/react";

export interface UseMagneticOptions {
  strength?: number;
  stiffness?: number;
  damping?: number;
}

export function useMagnetic(options: UseMagneticOptions = {}) {
  const { strength = 30, stiffness = 150, damping = 15 } = options;

  const ref = React.useRef<HTMLElement>(null);
  const x = useMotionValue(0);
  const y = useMotionValue(0);

  const springConfig = { damping, stiffness };
  const springX = useSpring(x, springConfig);
  const springY = useSpring(y, springConfig);

  const handleMouseMove = (e: React.MouseEvent<HTMLElement>) => {
    if (!ref.current) return;

    const rect = ref.current.getBoundingClientRect();
    const centerX = rect.left + rect.width / 2;
    const centerY = rect.top + rect.height / 2;

    const distanceX = e.clientX - centerX;
    const distanceY = e.clientY - centerY;

    x.set(distanceX * (strength / 100));
    y.set(distanceY * (strength / 100));
  };

  const handleMouseLeave = () => {
    x.set(0);
    y.set(0);
  };

  const magneticProps = {
    style: { x: springX, y: springY },
    onMouseMove: handleMouseMove,
    onMouseLeave: handleMouseLeave,
  };

  return { magneticProps, ref };
}
```

## Usage

```tsx
import { motion } from "motion/react"
import { useMagnetic } from "@/lib/hooks/use-magnetic"

export function MagneticButton() {
  const { magneticProps, ref } = useMagnetic()
  
  return (
    <motion.button 
      ref={ref}
      {...magneticProps}
      className="inline-flex h-11 items-center gap-2 rounded-lg bg-blue-600 px-6 text-sm font-semibold text-white"
    >
      Hover me - I follow your cursor!
    </motion.button>
  )
}
```

## Use cases

`useMagnetic` returns `{ magneticProps, ref }`. The `magneticProps` object contains a `style` with spring-driven `x` and `y` MotionValues, plus `onMouseMove` and `onMouseLeave` event handlers. Attach `ref` to the element so the hook can read its bounding rect.

- **Hero CTA buttons.** A primary action button that follows the cursor by a few pixels feels alive and invites a click more than a static button.
- **Social media icon links in a footer or profile page.** Small icons with `strength: 40` and `stiffness: 200` respond snappily to precise cursor movement.
- **Navigation pill links.** A subtle `strength: 20` on nav links creates a soft floating effect without moving the link far enough to confuse the user about its position.
- **Logo mark in a header.** A logo that gently shifts toward the cursor adds personality to an otherwise static header without interfering with the nav layout.
- **Floating action buttons.** An FAB with `strength: 35` and `damping: 12` gives a satisfying pull-toward-you feel before the user clicks.

## When to use / when not to use

**Use `useMagnetic` directly when:**

- You need the magnetic effect on a custom element, such as a `motion.a` or a styled icon wrapper, that is not covered by a prebuilt component.
- You want to compose the magnetic translation with another hook: because `magneticProps.style` sets `x` and `y` as MotionValues, you can pass those values into `useTransform` for derived effects.

**Avoid `useMagnetic` when:**

- The element is inside a scrollable container. The hook uses `getBoundingClientRect()` to compute the center; inside a scrolled container the `rect.left` and `rect.top` values are correct, but rapid scroll while hovering can cause visible jumps.
- Multiple adjacent elements each have the magnetic effect and their movement radius overlaps. Users hovering the gap between two magnetic elements can trigger both, making both elements shift unpredictably.
- The surface is touch-only. The `onMouseMove` handler never fires on a touch device, so the element appears completely static. Pair with [usePress](/docs/hooks/press) for touch feedback.
- The element is a form input or a text area. Mouse-following translation on an input moves the entire field, making it hard to click a specific character position.

## Accessibility notes

`useMagnetic` attaches `onMouseMove` and `onMouseLeave` event handlers that animate `x` and `y` via MotionValues. These are pointer-driven and have no effect on keyboard navigation.

- The magnetic effect is decorative. Do not use it to signal interactivity for keyboard users. Rely on semantic HTML (`<button>`, `<a>`) and visible `:focus-visible` styles for keyboard affordance.
- Respect `prefers-reduced-motion`. The spring-based motion can cause vestibular discomfort, especially with high `strength` values. When the preference is set, return static props with no movement:

```tsx
"use client"
import { motion } from "motion/react"
import { useReducedMotion } from "motion/react"
import { useMagnetic } from "@/lib/use-magnetic"

export function MagneticCTA({ label }: { label: string }) {
  const reduced = useReducedMotion()
  const { magneticProps, ref } = useMagnetic({ strength: 30 })

  return (
    <motion.button
      ref={ref}
      {...(reduced ? {} : magneticProps)}
      className="rounded-xl bg-blue-600 px-8 py-3 text-sm font-semibold text-white focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-blue-500"
    >
      {label}
    </motion.button>
  )
}
```

- Because the element translates away from its original position, ensure that its hover hitbox remains consistent. Framer Motion keeps the layout anchor at the original position, so the painted element may appear to drift outside its clickable area at large `strength` values (above 50). Keep `strength` at or below 40 for buttons that need precise clicking.

## Performance considerations

- `useMagnetic` drives `x` and `y` via `useMotionValue` and `useSpring`. Both are MotionValues that bypass React renders entirely. The spring interpolation is performed in Framer Motion's internal rAF loop and only updates the DOM via direct style writes. No component re-render occurs on mouse move.
- `onMouseMove` fires on every pointer move event. Framer Motion's MotionValue batches updates in its animation frame loop, so even rapid cursor movement does not schedule extra frames.
- Each `useMagnetic` call creates two springs. On pages with many magnetic elements, the total number of active springs grows linearly. Keep the number of simultaneously visible magnetic elements low (single digits) to avoid saturating the animation frame budget.
- When the element unmounts, Framer Motion automatically cancels the springs and removes the listeners. No manual cleanup is needed.

### Social links row in a footer

```tsx
"use client"
import { motion } from "motion/react"
import { useMagnetic } from "@/lib/use-magnetic"

const socials = [
  { href: "https://github.com", label: "GitHub", icon: "github" },
  { href: "https://twitter.com", label: "Twitter", icon: "twitter" },
  { href: "https://linkedin.com", label: "LinkedIn", icon: "linkedin" },
]

function SocialLink({ href, label }: { href: string; label: string }) {
  const { magneticProps, ref } = useMagnetic({ strength: 35, stiffness: 200, damping: 14 })
  return (
    <motion.a
      href={href}
      ref={ref}
      {...magneticProps}
      aria-label={label}
      className="flex h-9 w-9 items-center justify-center rounded-full border border-neutral-200 text-neutral-500 hover:border-neutral-400 hover:text-neutral-800 dark:border-neutral-700"
    >
      <span className="text-xs font-bold">{label[0]}</span>
    </motion.a>
  )
}

export function SocialRow() {
  return (
    <div className="flex gap-3">
      {socials.map((s) => <SocialLink key={s.href} href={s.href} label={s.label} />)}
    </div>
  )
}
```
