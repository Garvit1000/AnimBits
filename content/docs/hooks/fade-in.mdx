## Overview

The `useFadeIn` hook creates a smooth fade-in entrance animation. One of the most versatile and subtle animations, perfect for page loads, modal entrances, content reveals, and progressive disclosure.

<Callout type="info">
  **Subtle and elegant:** Fade-in is the most universally accepted animation, working well in any context.
</Callout>

import { type MotionProps, type Easing } from "motion/react";

export interface UseFadeInOptions {
  duration?: number;

  delay?: number;

  from?: number;

  to?: number;

  ease?: Easing | Easing[];

  viewport?: "once" | "always";
}

export function useFadeIn(options: UseFadeInOptions = {}): MotionProps {
  const {
    duration = 0.5,
    delay = 0,
    from = 0,
    to = 1,
    ease = "easeOut",
    viewport = "once",
  } = options;

  return {
    initial: {
      opacity: from,
    },
    whileInView: {
      opacity: to,
    },
    viewport: {
      once: viewport === "once",
    },
    transition: {
      duration,
      delay,
      ease,
    },
  };
}

### Returns

Returns `MotionProps` object containing:
- `initial` - Initial opacity state (0)
- `whileInView` - Target opacity state (1)
- `viewport` - Viewport animation config
- `transition` - Animation transition config

## Composing with Other Hooks

Combine fade with other animations:

```tsx
import { useFadeIn } from "@/lib/hooks/use-fade-in"
import { useSlideIn } from "@/lib/hooks/use-slide-in"

// Fade + Slide
const fade = useFadeIn({ duration: 0.6 })
const slide = useSlideIn({ direction: "left", distance: 30 })

const combined = {
  initial: {
    ...fade.initial,
    ...slide.initial,
  },
  whileInView: {
    ...fade.whileInView,
    ...slide.whileInView,
  },
  transition: fade.transition,
}

<motion.div {...combined}>Fade + Slide</motion.div>
```

## Use cases

`useFadeIn` returns `MotionProps` with `initial`, `whileInView`, `viewport`, and `transition` fields. Spread them on any `motion.*` element and the element fades in as it enters the viewport.

- **Hero sections and above-the-fold headings.** A `motion.section` or `motion.h1` with `useFadeIn({ duration: 0.8 })` announces content without jarring the user on page load.
- **Image galleries.** Each `motion.img` can receive `useFadeIn({ delay: index * 0.08 })` to stagger the reveal as the user scrolls down a grid.
- **Feature sections in a landing page.** Apply the hook to alternating `motion.article` or `motion.div` columns with `direction`-specific stagger delays.
- **Modal overlays.** Mount a `motion.div` backdrop with `useFadeIn({ duration: 0.2, viewport: "always" })` so the overlay fades in every time the modal opens, not just the first time.
- **Lazy-loaded content panels.** Sections that fetch data on scroll can fade in their content once it resolves, hiding the empty skeleton state transition.

## When to use / when not to use

**Use `useFadeIn` directly when:**

- The element you want to animate is not a standard prebuilt component, for example a `motion.aside` sidebar or a custom `motion.figure`.
- You want to combine the fade with another hook such as [useSlideIn](/docs/hooks/slide-in): merge the two `MotionProps` objects by spreading `initial` and `whileInView` from both.
- You need `viewport: "always"` so the element fades in every time it enters the viewport, not just the first.

**Avoid `useFadeIn` when:**

- The wait would be over 2 seconds. A slow fade delays content visibility and can frustrate users on slow connections.
- You are fading in text that is part of the primary content hierarchy. Screen readers render all text regardless of opacity, but sighted users may be confused if important text is invisible for a perceptible moment.
- You need a combined entrance with translation. Use [useSlideIn](/docs/hooks/slide-in) instead, which handles both opacity and translate together, or compose both hooks manually.

## Accessibility notes

`useFadeIn` uses `whileInView` and animates only `opacity`. The element exists in the DOM from mount and is accessible to screen readers even while invisible. However, there are still considerations:

Honor the user's reduced-motion preference by returning plain props with no animation:

```tsx
"use client"
import { motion } from "motion/react"
import { useReducedMotion } from "motion/react"
import { useFadeIn } from "@/lib/use-fade-in"

export function FeatureCard({ title, body }: { title: string; body: string }) {
  const reduced = useReducedMotion()
  const fadeProps = useFadeIn({ duration: 0.6 })

  return (
    <motion.div
      {...(reduced ? {} : fadeProps)}
      className="rounded-xl border border-neutral-200 p-6 dark:border-neutral-800"
    >
      <h3 className="font-semibold">{title}</h3>
      <p className="mt-2 text-sm text-neutral-500">{body}</p>
    </motion.div>
  )
}
```

Additional guidance:

- Do not fade in interactive elements such as buttons or links unless they become fully opaque before the user can reach them via keyboard Tab navigation. If they can be focused while still semi-transparent, add `pointer-events: none` during the animation.
- The default `from: 0` means the element starts fully transparent. Any content inside is still in the DOM, so `Ctrl+F` in-browser search and screen readers can still find it.
- Avoid `viewport: "always"` on sections that contain forms or other interactive controls. Reflowing the opacity on every scroll can momentarily hide UI state.

## Performance considerations

- `useFadeIn` animates only `opacity`, which is a compositor property. No layout or paint is triggered, and the animation runs off the main thread.
- The hook uses `whileInView`, which relies on an `IntersectionObserver` under the hood in Framer Motion. This is far cheaper than a scroll event listener and does not block the main thread.
- With `viewport: "once"` (the default), the observer disconnects after the first intersection, eliminating any ongoing cost.
- Stagger patterns calling `useFadeIn` inside a `.map()` create one `IntersectionObserver` per element. For very large lists (50 or more items) consider batching with Framer Motion's `staggerChildren` inside a `variants` object instead.
- `opacity` transitions do not affect `will-change`. You do not need to add `will-change: opacity` manually; Framer Motion handles compositing hints internally.

## Customization examples

### Staggered feature list with varying delays

```tsx
import { motion } from "motion/react"
import { useFadeIn } from "@/lib/use-fade-in"

const features = ["Type-safe API", "Tree-shakeable", "Zero config", "SSR ready"]

export function FeatureList() {
  return (
    <ul className="space-y-3">
      {features.map((feature, i) => {
        const fadeProps = useFadeIn({ delay: i * 0.1, duration: 0.5 })
        return (
          <motion.li key={feature} {...fadeProps} className="flex items-center gap-2 text-sm">
            <span className="h-1.5 w-1.5 rounded-full bg-blue-500" />
            {feature}
          </motion.li>
        )
      })}
    </ul>
  )
}
```

### Fade in combined with slide (manual composition)

```tsx
import { motion } from "motion/react"
import { useFadeIn } from "@/lib/use-fade-in"
import { useSlideIn } from "@/lib/use-slide-in"

export function SlideAndFadeCard() {
  const fade = useFadeIn({ duration: 0.5 })
  const slide = useSlideIn({ direction: "up", distance: 24, duration: 0.5 })

  const combined = {
    initial: { ...fade.initial, ...slide.initial },
    whileInView: { ...fade.whileInView, ...slide.whileInView },
    viewport: fade.viewport,
    transition: fade.transition,
  }

  return (
    <motion.div {...combined} className="rounded-xl border border-neutral-200 p-6">
      Slides up and fades in simultaneously
    </motion.div>
  )
}
```

### Fade that replays on every scroll into view

```tsx
import { motion } from "motion/react"
import { useFadeIn } from "@/lib/use-fade-in"

export function ReplaySection() {
  const fadeProps = useFadeIn({ viewport: "always", duration: 0.6 })

  return (
    <motion.section {...fadeProps} className="py-16">
      <h2 className="text-2xl font-bold">This section fades in every time</h2>
    </motion.section>
  )
}
```

const images = [
