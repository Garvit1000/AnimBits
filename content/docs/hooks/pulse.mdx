## Overview

The `usePulse` hook creates a continuous pulsing animation, perfect for notifications, badges, or drawing attention to important elements. The animation loops infinitely with smooth easing.

<Callout type="info">
  **Attention-grabbing:** Use pulse animations sparingly for truly important elements to avoid overwhelming users.
</Callout>

## Installation

<Tabs items={['Manual', 'CLI']}>
  <Tab value="Manual">
    Copy and paste the hook code into your project:

    ```tsx title="lib/hooks/use-pulse.ts"
    import { type MotionProps } from "framer-motion"

    export interface UsePulseOptions {
      scaleFrom?: number
      scaleTo?: number
      duration?: number
      ease?: string | number[]
      repeat?: number
      delay?: number
    }

    export function usePulse(options: UsePulseOptions = {}): MotionProps {
      const {
        scaleFrom = 1,
        scaleTo = 1.05,
        duration = 1,
        ease = "easeInOut",
        repeat = Infinity,
        delay = 0,
      } = options

      return {
        animate: {
          scale: [scaleFrom, scaleTo, scaleFrom],
        },
        transition: {
          duration,
          ease,
          repeat,
          delay,
        },
      }
    }
    ```
  </Tab>

## Usage


### Returns

Returns `MotionProps` object containing:
- `animate` - Animation keyframes for continuous pulsing
- `transition` - Animation transition config with repeat

## Accessibility

<Callout type="warn">
  **Motion sensitivity:** Continuous animations can be distracting or cause discomfort for some users. Consider respecting `prefers-reduced-motion`:
  
  ```tsx
  const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches
  const pulseProps = prefersReducedMotion ? {} : usePulse()
  ```
</Callout>

- Use pulse animations sparingly
- Avoid pulsing large elements
- Provide alternative indicators for important information
- Consider adding a pause/stop option for continuous animations

## Browser Support

Works in all modern browsers that support CSS transforms and Framer Motion:
- Chrome/Edge 90+
- Firefox 88+
- Safari 14+

## Performance Tips

- Pulse animations are GPU-accelerated via CSS transforms
- Avoid pulsing too many elements simultaneously
- Use `will-change: transform` for better performance on complex layouts
