## Overview

The `useBounce` hook creates a bouncing animation effect. Use it continuously for attention-grabbing elements or on hover for interactive feedback. Perfect for playful UIs, celebrations, and notifications.

import { type MotionProps, type Easing } from "motion/react";

export interface UseBounceOptions {
  bounceHeight?: number;

  duration?: number;

  ease?: Easing | Easing[];

  repeat?: number;

  delay?: number;

  onHover?: boolean;
}

export function useBounce(options: UseBounceOptions = {}): MotionProps {
  const {
    bounceHeight = -20,
    duration = 0.6,
    ease = [0.68, -0.55, 0.265, 1.55],
    repeat = Infinity,
    delay = 0,
    onHover = false,
  } = options;

  if (onHover) {
    return {
      whileHover: {
        y: [0, bounceHeight, 0],
      },
      transition: {
        duration,
        ease,
      },
    };
  }

  return {
    animate: {
      y: [0, bounceHeight, 0],
    },
    transition: {
      duration,
      ease,
      repeat,
      delay,
    },
  };
}

## Usage


### Bounce on Hover

```tsx
import { motion } from "motion/react"
import { useBounce } from "@/lib/hooks/use-bounce"

export function HoverBounce() {
  const bounceProps = useBounce({ onHover: true })
  
  return (
    <motion.button {...bounceProps} className="btn">
      Hover me!
    </motion.button>
  )
}
```



### Limited Bounces

```tsx
import { motion } from "motion/react"
import { useBounce } from "@/lib/hooks/use-bounce"

export function ThreeBounces() {
  const bounceProps = useBounce({ 
    repeat: 3,  // Bounce 3 times then stop
    bounceHeight: -25 
  })
  
  return (
    <motion.div {...bounceProps}>
      Bounces 3 times
    </motion.div>
  )
}
```

## API Reference

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `bounceHeight` | `number` | `-20` | Bounce height in pixels (negative = up) |
| `duration` | `number` | `0.6` | Duration for one bounce cycle |
| `ease` | `string \| number[]` | `[0.68, -0.55, 0.265, 1.55]` | Bounce easing (cubic bezier) |
| `repeat` | `number` | `Infinity` | Number of times to repeat |
| `delay` | `number` | `0` | Delay before animation starts |
| `onHover` | `boolean` | `false` | Bounce on hover instead of continuous |

## Use cases

`useBounce` is component-agnostic: the returned `MotionProps` can be spread onto any `motion.*` element.

- **Notification badges and counters.** A small badge that bounces continuously draws the eye to new unread items without requiring the user to hunt for them.
- **Scroll-down indicators.** Spread the props onto a `motion.div` wrapping an arrow icon. Use `bounceHeight: 10` (positive, so it moves downward) to point the user toward more content.
- **Hover-activated icon buttons.** Pass `onHover: true` to a `motion.button` wrapping a social icon or action button. The bounce starts only on pointer entry, keeping the idle state clean.
- **Success or completion feedback.** After a form submits, mount a checkmark inside a `motion.div` with `repeat: 3` so it bounces three times and stops, signaling completion without looping forever.
- **Playful empty states.** A bouncing illustration in an empty-state screen adds personality without being permanently distracting.

## When to use / when not to use

**Use `useBounce` when:**

- You need the effect on a custom element that is not covered by a prebuilt component, such as a `motion.span` inside a heading or a `motion.li` in a custom list.
- You want to compose bouncing with another hook, for example combining it with `usePress` so an icon button bounces on hover and shrinks on click.
- You need fine control over `bounceHeight`, `repeat`, or `ease` that the prebuilt component does not expose.

**Reach for the prebuilt component when:**

- You just need a standard button or badge with a bounce and do not need to customize the host element or compose effects.

**Avoid `useBounce` when:**

- The element is large (a card or a section). A full-card bounce is disorienting; prefer [useHoverLift](/docs/hooks/hover-lift) for card-level hover feedback.
- Multiple elements bounce simultaneously. Stagger the `delay` option or switch to a one-time `repeat: 1` to avoid overwhelming the user.
- The bounce carries critical information by itself, such as "there are new items." Always pair it with a text label or badge count so non-animated experiences remain informative.

## Accessibility notes

The bounce animation animates the `y` transform, which is composited and does not affect the DOM structure or ARIA tree. However, perpetual motion can be distracting or cause vestibular discomfort.

Respect the user's reduced-motion preference by returning empty props when the OS setting is active:

```tsx
"use client"
import { motion } from "motion/react"
import { useReducedMotion } from "motion/react"
import { useBounce } from "@/lib/use-bounce"

export function NotificationBadge({ count }: { count: number }) {
  const reduced = useReducedMotion()
  const bounceProps = useBounce({ bounceHeight: -12, duration: 0.5 })

  return (
    <motion.span
      {...(reduced ? {} : bounceProps)}
      className="inline-flex h-5 w-5 items-center justify-center rounded-full bg-red-500 text-xs font-bold text-white"
    >
      {count}
    </motion.span>
  )
}
```

Additional guidance:

- When `onHover: true` is used on a `motion.button`, the bounce is triggered only by pointer interaction and does not run continuously, which is friendlier for motion-sensitive users.
- Do not rely on the bounce animation alone to convey meaning. Pair it with visible text or an icon so keyboard-only and screen reader users receive the same information.
- If the element wraps interactive content, ensure focus styles remain visible. Framer Motion's `whileHover` and `animate` do not remove focus outlines, but verify your CSS does not suppress them.

## Performance considerations

- `useBounce` animates only the `y` transform via the `animate` or `whileHover` props. Transforms run on the compositor thread and do not trigger layout or paint, so the animation stays at 60fps even on modest hardware.
- With the default `repeat: Infinity`, the animation keeps the compositor awake indefinitely. Conditionally render the element when the bounce is no longer needed, or use `repeat: 3` for a finite number of bounces, so the compositor thread is freed after the last cycle.
- When dozens of elements on a single page use `repeat: Infinity`, each schedules its own frame. Limit continuous bouncing to one or two focal elements per view.
- The custom cubic-bezier easing `[0.68, -0.55, 0.265, 1.55]` produces the overshoot that makes the bounce feel springy. It is evaluated in JavaScript by Framer Motion but has negligible CPU cost.

## Real-world examples

### Unread notification badge in a nav bar

```tsx
"use client"
import { motion } from "motion/react"
import { useReducedMotion } from "motion/react"
import { useBounce } from "@/lib/use-bounce"

export function NavBell({ unread }: { unread: number }) {
  const reduced = useReducedMotion()
  const bounceProps = useBounce({ bounceHeight: -10, duration: 0.5 })

  return (
    <button className="relative p-2 text-neutral-600 hover:text-neutral-900" aria-label="Notifications">
      <svg width="20" height="20" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="2">
        <path d="M18 8A6 6 0 0 0 6 8c0 7-3 9-3 9h18s-3-2-3-9" />
        <path d="M13.73 21a2 2 0 0 1-3.46 0" />
      </svg>
      {unread > 0 && (
        <motion.span
          {...(reduced ? {} : bounceProps)}
          className="absolute right-1 top-1 inline-flex h-4 w-4 items-center justify-center rounded-full bg-red-500 text-[10px] font-bold text-white"
        >
          {unread}
        </motion.span>
      )}
    </button>
  )
}
```
