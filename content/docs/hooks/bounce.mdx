## Overview

The `useBounce` hook creates a bouncing animation effect. Use it continuously for attention-grabbing elements or on hover for interactive feedback. Perfect for playful UIs, celebrations, and notifications.

<Callout type="info">
  **Playful interactions:** Bounce animations add personality and energy to your interface.
</Callout>

## Installation

<Tabs items={['Manual', 'CLI']}>
  <Tab value="Manual">
    ```tsx title="lib/hooks/use-bounce.ts"
    import { type MotionProps } from "framer-motion"

    export interface UseBounceOptions {
      bounceHeight?: number
      duration?: number
      ease?: string | number[]
      repeat?: number
      delay?: number
      onHover?: boolean
    }

    export function useBounce(options: UseBounceOptions = {}): MotionProps {
      const {
        bounceHeight = -20,
        duration = 0.6,
        ease = [0.68, -0.55, 0.265, 1.55],
        repeat = Infinity,
        delay = 0,
        onHover = false,
      } = options

      if (onHover) {
        return {
          whileHover: {
            y: [0, bounceHeight, 0],
          },
          transition: {
            duration,
            ease,
          },
        }
      }

      return {
        animate: {
          y: [0, bounceHeight, 0],
        },
        transition: {
          duration,
          ease,
          repeat,
          delay,
        },
      }
    }
    ```

## API Reference

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `bounceHeight` | `number` | `-20` | Bounce height in pixels (negative = up) |
| `duration` | `number` | `0.6` | Duration for one bounce cycle |
| `ease` | `string \| number[]` | `[0.68, -0.55, 0.265, 1.55]` | Bounce easing (cubic bezier) |
| `repeat` | `number` | `Infinity` | Number of times to repeat |
| `delay` | `number` | `0` | Delay before animation starts |
| `onHover` | `boolean` | `false` | Bounce on hover instead of continuous |

## Accessibility

- Continuous bounce animations can be distracting
- Consider using `onHover: true` for better UX
- Respect `prefers-reduced-motion` for accessibility
- Use sparingly for important elements only

## Browser Support

Works in all modern browsers that support CSS transforms and Framer Motion:
- Chrome/Edge 90+
- Firefox 88+
- Safari 14+
