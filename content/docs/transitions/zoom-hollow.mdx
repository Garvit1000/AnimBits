## Overview

The `ZoomHollowText` component creates a stunning text transition with zoom-in animation and hollow outline effect. Perfect for hero sections, theme toggles, and attention-grabbing headlines. Works seamlessly with light and dark modes.

    ```tsx title="components/animations/transitions/zoom-hollow.tsx"
    "use client";

    import React, { useState, useEffect } from "react";
    import { motion, AnimatePresence } from "motion/react";
    import { cn } from "@/lib/utils";

    interface ZoomHollowTextProps {
      words: string[];
      className?: string;
      duration?: number;
      interval?: number;
      strokeWidth?: number;
      fillColor?: string;
      strokeColor?: string;
    }

    export function ZoomHollowText({
      words,
      className = "",
      duration = 0.8,
      interval = 2500,
      strokeWidth = 2,
      fillColor = "transparent",
      strokeColor = "currentColor",
    }: ZoomHollowTextProps) {
      const [index, setIndex] = useState(0);

      useEffect(() => {
        const timer = setInterval(() => {
          setIndex((prev) => (prev + 1) % words.length);
        }, interval);
        return () => clearInterval(timer);
      }, [words.length, interval]);

      return (
        <div className={cn("relative inline-block", className)}>
          <AnimatePresence mode="wait">
            <motion.span
              key={words[index]}
              initial={{ scale: 0.5, opacity: 0, filter: "blur(10px)" }}
              animate={{ scale: 1, opacity: 1, filter: "blur(0px)" }}
              exit={{ scale: 1.5, opacity: 0, filter: "blur(10px)" }}
              transition={{
                duration,
                ease: [0.22, 1, 0.36, 1],
              }}
              style={{
                WebkitTextStroke: `${strokeWidth}px ${strokeColor}`,
                WebkitTextFillColor: fillColor,
                paintOrder: "stroke fill",
              }}
              className="block"
            >
              {words[index]}
            </motion.span>
          </AnimatePresence>
        </div>
      );
    }
    ```

## CSS Customization

The hollow effect is created using webkit text stroke. You can customize it with CSS:

```css
.custom-hollow {
  -webkit-text-stroke: 3px currentColor;
  -webkit-text-fill-color: transparent;
  paint-order: stroke fill;
}
```

For dark mode support, use CSS variables:

```css
.theme-aware-hollow {
  -webkit-text-stroke: 2px hsl(var(--foreground));
  -webkit-text-fill-color: transparent;
}

```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `words` | `string[]` | Required | Array of words to cycle through |
| `className` | `string` | `""` | Additional CSS classes |
| `duration` | `number` | `0.8` | Animation duration in seconds |
| `interval` | `number` | `2500` | Time between word changes in ms |
| `strokeWidth` | `number` | `2` | Width of the text outline |
| `fillColor` | `string` | `"transparent"` | Fill color of the text |
| `strokeColor` | `string` | `"currentColor"` | Color of the text outline |

## Use cases

`ZoomHollowText` cycles through `words` on a `setInterval` every `interval` milliseconds (default 2500ms). Each word change uses `AnimatePresence mode="wait"`: the current word exits (`scale: 1.5`, `opacity: 0`, `filter: blur(10px)`), then the next word enters (`scale: 0.5` to `scale: 1`, `opacity: 0` to `1`, blur clears). The custom easing `[0.22, 1, 0.36, 1]` (a fast ease-out curve) makes the entry feel snappy and the exit feel like the word explodes outward. The hollow outline is CSS-only via `-webkit-text-stroke` and `-webkit-text-fill-color: transparent`.

Use it when:

- **Hero sections.** A headline like "We help you [Design / Build / Ship / Scale]" where the cycling word reinforces the brand message. The hollow outline at large sizes (`text-7xl` or above) has high visual impact.
- **Value proposition cycling.** A landing page that cycles through product benefits. `interval=3000` with `duration=0.8` gives enough reading time between transitions.
- **Loading or splash screens.** While a heavy page loads, a cycling word display entertains the user. `interval=1500` with `duration=0.5` keeps energy high.
- **Theme-aware decorative text.** With `strokeColor="currentColor"` and `fillColor="transparent"`, the outline color automatically matches the current text color in both light and dark mode, requiring no custom CSS.

## When to use / when not to use

**Use `ZoomHollowText` when:**

- The words are short (1 to 2 words, ideally under 12 characters). Long words at `text-7xl` will overflow their container on mobile.
- The context is decorative, not informational. Cycling text is not readable for users who need to parse it to understand the page.
- You control when the component is visible. Running `setInterval` continuously when the element is off-screen wastes CPU. Use an `IntersectionObserver` to pause cycling when scrolled away.

**Do not use `ZoomHollowText` when:**

- The cycling message is essential for the user to read before taking an action. `AnimatePresence mode="wait"` means only one word is in the DOM at a time; users who look away will miss words.
- The words have very different lengths. `motion.span` has `display: block`, so its bounding box changes with each word. The parent container must have a fixed height to prevent layout shift.
- You need the animation to stop at a specific word (e.g., after user interaction). The component has no `pause` or `index` prop; pausing requires lifting the `index` state out of the component.

## Accessibility notes

- **Decorative content.** The cycling text conveys no critical information. Wrap it in a container with `aria-hidden="true"` and provide the primary message in a static, always-visible sibling element. This is the safest approach.
- **If the text is meaningful.** If the cycling words communicate important product benefits, add a visually-hidden list of all words so screen readers read the complete message without the animation. Example: `<ul className="sr-only"><li>Design</li><li>Create</li><li>Innovate</li></ul>`.
- **`prefers-reduced-motion`.** The component does not currently check `useReducedMotion`. Add it to skip the enter/exit animation and display words statically, just updating the text without the zoom-blur transition:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { ZoomHollowText } from "@/components/animations/transitions/zoom-hollow"

// Static fallback when the user prefers reduced motion.
function AccessibleZoomHollow(props: React.ComponentProps<typeof ZoomHollowText>) {
  const reduced = useReducedMotion()
  if (reduced) {
    return (
      <span style={{ WebkitTextStroke: `${props.strokeWidth ?? 2}px ${props.strokeColor ?? "currentColor"}`, WebkitTextFillColor: props.fillColor ?? "transparent" }}>
        {props.words[0]}
      </span>
    )
  }
  return <ZoomHollowText {...props} />
}
```

- **`-webkit-text-stroke` support.** `-webkit-text-stroke` and `-webkit-text-fill-color` are supported in all modern browsers but are technically prefixed. They have no standard fallback in Firefox for very old versions. Test and provide a `fillColor="currentColor"` fallback if full-text support is required.

## Performance considerations

- **`AnimatePresence mode="wait"`.** Only one `motion.span` is in the DOM at a time. The exiting word unmounts before the entering word mounts. This keeps the DOM clean but means two full enter/exit animations fire on each cycle. The combined animation cost is `duration * 2` per cycle (the exit overlaps with the enter by `mode="wait"` semantics, actually it is sequential, so cost is the exit duration plus the enter duration).
- **Blur in `filter`.** The component animates `filter: blur(10px)` on enter and exit. CSS `filter` creates a new stacking context. For a small inline text span this is cheap, but if the span is very large (e.g., a massive `text-9xl` hero headline), the blur area is large and can cause GPU overdraw. Test on mobile and reduce `blur` if needed by using `duration=0.5` with no blur in the transition config (currently hardcoded at 10px; to change it you would need to modify the component).
- **`setInterval` + `setIndex`.** Each interval tick calls `setIndex`, which re-renders the component. This is a minimal re-render (only the `motion.span` changes), but keep the component's parent tree shallow to avoid triggering expensive ancestor re-renders.
- **Custom easing `[0.22, 1, 0.36, 1]`.** This is a fast cubic-bezier with an overshooting feel at the end of the `scale` animation on enter. It is specified as an array, which Framer Motion evaluates as a cubic-bezier. There is no spring settling involved, so the animation ends at exactly `duration` seconds.
