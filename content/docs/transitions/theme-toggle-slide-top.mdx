## Overview

The `ThemeToggleSlideTop` component creates a slide-down effect from the top when toggling themes. Uses the native View Transitions API with customizable speed and blur.

<Callout type="info">
  **Browser Support:** View Transitions API is supported in Chrome 111+, Edge 111+, and Opera 97+. Falls back to instant theme changes in unsupported browsers.
</Callout>

    ```tsx title="components/animations/transitions/theme-toggle-slide-top.tsx"
"use client";
import * as React from "react";
export interface ThemeToggleSlideTopProps {
  children: React.ReactNode;
  onToggle?: () => void;
  theme?: "light" | "dark";
  className?: string;
  speed?: number;
  blur?: number;
}
export function ThemeToggleSlideTop({
  children,
  onToggle,
  theme,
  className,
  speed = 0.5,
  blur = 0,
}: ThemeToggleSlideTopProps) {
  const [isTransitioning, setIsTransitioning] = React.useState(false);
  const handleClick = async () => {
    if (isTransitioning) return;
    if (!document.startViewTransition) {
      onToggle?.();
      return;
    }
    setIsTransitioning(true);
    document.documentElement.style.setProperty(
      "--transition-speed",
      `${speed}s`,
    );
    document.documentElement.style.setProperty(
      "--transition-blur",
      `${blur}px`,
    );
    document.documentElement.classList.add("theme-slide-top");
    const transition = document.startViewTransition(() => {
      onToggle?.();
    });
    try {
      await transition.finished;
    } catch (error) {
      console.warn("Theme transition interrupted:", error);
    } finally {
      document.documentElement.classList.remove("theme-slide-top");
      setIsTransitioning(false);
    }
  };
  return (
    <div
      onClick={handleClick}
      className={className}
      style={{ pointerEvents: isTransitioning ? "none" : "auto" }}
    >
      {children}
    </div>
  );
}

```

    **Required CSS** (add to `globals.css`):
    ```css
    :root {
      --transition-speed: 0.5s;
      --transition-blur: 0px;
    }

    .theme-slide-top::view-transition-old(root) {
      animation: slide-out-top var(--transition-speed) cubic-bezier(0.4, 0, 0.2, 1);
      filter: blur(var(--transition-blur));
    }

    .theme-slide-top::view-transition-new(root) {
      animation: slide-in-top var(--transition-speed) cubic-bezier(0.4, 0, 0.2, 1);
      filter: blur(var(--transition-blur));
    }

    @keyframes slide-out-top {
      to {
        transform: translateY(-100%);
      }
    }

    @keyframes slide-in-top {
      from {
        transform: translateY(-100%);
      }
    }

    @media (prefers-reduced-motion: reduce) {
      .theme-slide-top::view-transition-group(*),
      .theme-slide-top::view-transition-old(*),
      .theme-slide-top::view-transition-new(*) {
        animation: none !important;
      }
    }
    ```

## Usage

### With next-themes

```tsx
"use client"

import { useTheme } from "next-themes"
import { ThemeToggleSlideTop } from "@/components/animations/transitions/theme-toggle-slide-top"
import { Button } from "@/components/ui/button"

export function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  
  return (
    <ThemeToggleSlideTop 
      onToggle={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
      speed={0.5}
      blur={0}
    >
      <Button>Toggle Theme</Button>
    </ThemeToggleSlideTop>
  )
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The trigger element (button, icon, etc.) |
| `onToggle` | `() => void` | - | Callback when theme should toggle |
| `theme` | `"light"` \| `"dark"` | - | Current theme (optional) |
| `speed` | `number` | `0.5` | Animation speed in seconds |
| `blur` | `number` | `0` | Blur amount in pixels |
| `className` | `string` | - | Additional CSS classes |

## Use cases

`ThemeToggleSlideTop` adds `theme-slide-top` to `<html>`, then calls `document.startViewTransition`. The CSS animates `slide-out-top` (`translateY(-100%)` exit) and `slide-in-top` (`translateY(-100%)` enter) using `cubic-bezier(0.4, 0, 0.2, 1)`. The current page slides upward and off; the new theme slides down from the top to fill it.

Use it when:

- Your theme toggle is in a top navigation bar or a sticky header. The new theme cascading down from the top reads as "coming from" the header where the toggle lives.
- The interface has a vertical information hierarchy (hero at top, content below) and the top-down wipe reinforces that hierarchy.
- You want a "curtain drop" feel: the new theme descends like a curtain from the navbar, which is a familiar metaphor for a mode change.

## When to use / when not to use

**Use slide-top when:**

- The toggle is in a sticky top navbar, a desktop app's top menu bar, or a header announcement bar. The downward reveal connects spatially to the control.
- You want a "closing curtain" metaphor where the new theme is revealed as the old one rises out of view.

**Consider a different variant when:**

- The toggle is in a bottom nav bar. [Slide Bottom](/docs/transitions/theme-toggle-slide-bottom) (upward reveal) is more spatially coherent.
- The toggle is in a left or right sidebar. [Slide Left](/docs/transitions/theme-toggle-slide-left) or [Slide Right](/docs/transitions/theme-toggle-slide-right) respect the horizontal origin.
- You want the ripple to track the cursor position. [Circular](/docs/transitions/theme-toggle-circular) follows the exact click point regardless of where the button is.

**Do not use any theme toggle when:**

- `document.startViewTransition` is not available. All variants fall back to an instant theme swap via the `!document.startViewTransition` guard, which is the correct behavior.

## Accessibility notes

- **Keyboard accessibility.** The outer `<div onClick={...}>` is not keyboard-reachable. Always pass a focusable `<button>` element as `children`.
- **State communication.** Add `aria-pressed={isDark}` to the inner button (or `role="switch" aria-checked={isDark}`) so screen readers announce the current mode. The visual Sun/Moon icon swap is invisible to assistive technology.
- **Label.** An icon-only button must have `aria-label="Toggle theme"` or equivalent. Without it, screen readers may say "button" with no context.
- **`prefers-reduced-motion`.** The bundled CSS includes `@media (prefers-reduced-motion: reduce)` which sets `animation: none !important` for all `::view-transition-*` pseudo-elements when `.theme-slide-top` is present. No extra JavaScript is required; the theme change still fires, just without the slide animation.
- **`pointerEvents: none` window.** The wrapper is unclickable while `isTransitioning` is true. Keep `speed` at or below 0.6s to minimize this window.

## Performance considerations

- **Transform-only animation.** `slide-out-top` and `slide-in-top` animate only `translateY(-100%)`. This runs on the GPU compositor; no layout or paint is triggered.
- **No DOM reads on click.** The handler does not read `clientX`/`clientY`. It only writes CSS custom properties and adds a class, which are cheap DOM write operations.
- **`cubic-bezier(0.4, 0, 0.2, 1)` suitability.** This easing has a gentle ease-in followed by a pronounced ease-out. At `speed=0.5` it feels polished for a full-screen slide. At `speed=0.2` the ease-in portion compresses; consider overriding to plain `ease-out` for sub-0.3s transitions.
- **`blur` prop.** Applying `filter: blur()` to the full-viewport `::view-transition-old` and `::view-transition-new` layers is GPU-intensive. Restrict to desktop-class contexts and keep values at or below 8px.
- **Class lifecycle.** `theme-slide-top` is always removed in the `finally` block, so interrupted transitions do not leave stale state.

## Customization examples

### Top-to-bottom with faster ease

A crisper slide-down that emphasizes the new theme arriving:

```css
.theme-slide-top::view-transition-old(root) {
  animation: slide-out-top var(--transition-speed) ease-in;
}
.theme-slide-top::view-transition-new(root) {
  animation: slide-in-top var(--transition-speed) ease-out;
}
```

### Slow curtain with subtle blur

A cinematic top-down wipe for a portfolio or showcase site:

```tsx
<ThemeToggleSlideTop speed={0.9} blur={6} onToggle={toggleTheme}>
  <Button variant="ghost" size="icon" aria-label="Toggle theme">
    <Sun className="h-5 w-5 dark:hidden" />
    <Moon className="h-5 w-5 hidden dark:block" />
  </Button>
</ThemeToggleSlideTop>
```

## Real-world examples

### Top navbar theme toggle

The most common placement: a Sun/Moon icon button in a sticky header, with the new theme sliding down from where the button lives.

```tsx
"use client"
import { useTheme } from "next-themes"
import { ThemeToggleSlideTop } from "@/components/animations/transitions/theme-toggle-slide-top"
import { Moon, Sun } from "lucide-react"
import { Button } from "@/components/ui/button"

export function TopNavThemeToggle() {
  const { theme, setTheme } = useTheme()

  const toggleTheme = () => {
    const isDark = document.documentElement.classList.contains("dark")
    document.documentElement.classList.toggle("dark", !isDark)
    setTheme(isDark ? "light" : "dark")
  }

  return (
    <header className="sticky top-0 z-40 flex items-center justify-between px-6 h-14 border-b border-zinc-200 dark:border-zinc-800 bg-white/80 dark:bg-zinc-950/80 backdrop-blur">
      <span className="font-bold text-zinc-900 dark:text-zinc-100">Acme</span>
      <ThemeToggleSlideTop speed={0.5} onToggle={toggleTheme}>
        <Button variant="ghost" size="icon" aria-label="Toggle theme" aria-pressed={theme === "dark"}>
          <Sun className="h-5 w-5 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
          <Moon className="absolute h-5 w-5 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
        </Button>
      </ThemeToggleSlideTop>
    </header>
  )
}
```

### App menu bar toggle (desktop-class)

For a desktop-style app with a top menu bar, a slower speed with light blur gives a more "weighty" feel:

```tsx
"use client"
import { useTheme } from "next-themes"
import { ThemeToggleSlideTop } from "@/components/animations/transitions/theme-toggle-slide-top"

export function MenuBarTheme() {
  const { theme, setTheme } = useTheme()

  const toggleTheme = () => {
    const isDark = document.documentElement.classList.contains("dark")
    document.documentElement.classList.toggle("dark", !isDark)
    setTheme(isDark ? "light" : "dark")
  }

  return (
    <div className="flex items-center h-8 bg-zinc-100 dark:bg-zinc-900 border-b border-zinc-200 dark:border-zinc-800 px-4 gap-4">
      <span className="text-xs font-medium text-zinc-700 dark:text-zinc-300">File</span>
      <span className="text-xs font-medium text-zinc-700 dark:text-zinc-300">Edit</span>
      <span className="text-xs font-medium text-zinc-700 dark:text-zinc-300">View</span>
      <ThemeToggleSlideTop speed={0.55} blur={4} onToggle={toggleTheme}>
        <button
          aria-pressed={theme === "dark"}
          aria-label="Toggle theme"
          className="text-xs font-medium text-zinc-700 dark:text-zinc-300 hover:text-zinc-900 dark:hover:text-zinc-100"
        >
          Theme
        </button>
      </ThemeToggleSlideTop>
    </div>
  )
}
```
