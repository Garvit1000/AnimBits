## What is AnimBits?

AnimBits is a collection of **copy-paste animation components** and **primitive hooks** built on Framer Motion. Unlike traditional animation libraries, AnimBits gives you the source code. Customize everything to match your design system.

<Callout type="info">
  **Not a component library.** AnimBits is a collection of re-usable components and hooks that you can copy and paste into your apps. Built with React, TypeScript, and Framer Motion.
</Callout>

## Quick Start

<Steps>
  <Step>
    ### Install dependencies
    
    AnimBits requires Framer Motion and React. Install them if you haven't already:
    
    ```bash
    npm install framer-motion
    ```
  </Step>
  
  <Step>
    ### Copy a component or hook
    
    Browse the [components](/docs/components/buttons) or [hooks](/docs/hooks/hover-lift) and copy the code you need:
    
    <Tabs items={['Component', 'Hook']}>
      <Tab value="Component">
        ```tsx
        // Copy from registry/buttons/lift.tsx
        import { LiftButton } from '@/components/animations/buttons/lift'
        
        export function MyComponent() {
          return (
            <LiftButton className="px-6 py-3 bg-blue-500 text-white rounded-lg">
              Hover me!
            </LiftButton>
          )
        }
        ```
      </Tab>
      
      <Tab value="Hook">
        ```tsx
        // Copy from lib/hooks/use-hover-lift.ts
        import { motion } from 'framer-motion'
        import { useHoverLift } from '@/lib/hooks/use-hover-lift'
        
        export function MyComponent() {
          const liftProps = useHoverLift({ liftDistance: 12 })
          
          return (
            <motion.div {...liftProps} className="p-6 bg-white rounded-lg">
              This card lifts on hover!
            </motion.div>
          )
        }
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step>
    ### Customize to your needs
    
    Modify the animations, change the styling, add your own features. It's your code now!
  </Step>
</Steps>

## Architecture

AnimBits uses a layered architecture that gives you flexibility in how you use it:

```
┌─────────────────────────────────────────┐
│ Layer 3: Semantic Components            │
│ (buttons/lift, cards/hover-glow, etc.)  │
│ Quick to use, opinionated examples      │
└─────────────────────────────────────────┘
                  ↓ uses
┌─────────────────────────────────────────┐
│ Layer 2: Generic Wrappers (Optional)    │
│ (<Animated />, merge utilities)         │
│ Convenience layer for common patterns   │
└─────────────────────────────────────────┘
                  ↓ uses
┌─────────────────────────────────────────┐
│ Layer 1: Animation Primitives           │
│ (useHoverLift, useGlow, useFadeIn...)   │
│ Component-agnostic, maximum flexibility │
└─────────────────────────────────────────┘
```

### Choose Your Layer

<Tabs items={['Semantic Components', 'Primitive Hooks', 'Generic Wrappers']}>
  <Tab value="Semantic Components">
    **Best for:** Quick prototyping, consistent patterns
    
    ```tsx
    import { LiftButton } from '@/components/animations/buttons/lift'
    
    <LiftButton>Click me</LiftButton>
    ```
    
    - Fastest to implement
    - Opinionated styling included
    - Less flexible for custom use cases
  </Tab>
  
  <Tab value="Primitive Hooks">
    **Best for:** Custom implementations, maximum flexibility
    
    ```tsx
    import { motion } from 'framer-motion'
    import { useHoverLift } from '@/lib/hooks/use-hover-lift'
    
    const liftProps = useHoverLift({ liftDistance: 16 })
    <motion.article {...liftProps}>
      Custom article with lift!
    </motion.article>
    ```
    
    - Works with ANY element
    - Easy to compose multiple animations
    - Component-agnostic
    - Requires more setup
  </Tab>
  
  <Tab value="Generic Wrappers">
    **Best for:** Consistent animation patterns across elements
    
    ```tsx
    import { Animated } from '@/lib/components/animated'
    import { useHoverLift } from '@/lib/hooks/use-hover-lift'
    
    const lift = useHoverLift()
    <Animated animation={lift} as="article">
      Convenient wrapper!
    </Animated>
    ```
    
    - Cleaner than raw motion components
    - Still fully flexible
    - Extra abstraction layer
  </Tab>
</Tabs>

## Philosophy

<Callout>
  **You own the code.** AnimBits is not a package you install. It's a collection of components and hooks you copy into your project. This means:
  
  - No version conflicts or breaking changes
  - Full control over implementation
  - Customize without fighting the library
  - Learn by reading the source
</Callout>

### Why Copy-Paste?

Traditional animation libraries force you into their patterns. AnimBits gives you **the building blocks** and gets out of your way. Want to modify the easing? Change the shadow? Combine three different animations? Go for it. It's your code.

## What's Included

- **8+ Primitive Hooks** — `useHoverLift`, `useHoverGlow`, `useFadeIn`, `useScaleIn`, `useSlideIn`, `usePulse`, `usePress`, `useBounce`
- **40+ Semantic Components** — Buttons, cards, icons, lists, pages, text, loaders, and transitions
- **TypeScript** — Fully typed with intelligent autocomplete
- **Composable** — Mix and match animations freely
- **Accessible** — Built with accessibility in mind
