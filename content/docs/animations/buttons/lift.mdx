## Overview

The `LiftButton` component creates a button that lifts up on hover with an optional shadow effect. Built using the `useHoverLift` primitive hook, providing a polished, interactive feel.

<Callout type="info">
  **Uses primitive hook:** This component uses `useHoverLift` internally. You can use that hook directly on any element for the same effect.
</Callout>

    **Step 1: Copy the hook**
    
    ```tsx title="lib/hooks/use-hover-lift.ts"
    import { type MotionProps, type Easing } from "motion/react";

    export interface UseHoverLiftOptions {
      liftDistance?: number;
      addShadow?: boolean;
      shadowIntensity?: number;
      duration?: number;
      ease?: Easing | Easing[];
      scale?: number;
    }

    export function useHoverLift(options: UseHoverLiftOptions = {}): MotionProps {
      const {
        liftDistance = 8,
        addShadow = true,
        shadowIntensity = 0.3,
        duration = 0.2,
        ease = "easeOut",
        scale = 1,
      } = options;

      const shadowBlur = liftDistance * 2;
      const shadowSpread = Math.round(liftDistance / 4);

      return {
        initial: {
          y: 0,
          scale: 1,
          boxShadow: addShadow
            ? `0 2px 4px rgba(0, 0, 0, ${shadowIntensity * 0.3})`
            : undefined,
        },
        whileHover: {
          y: -liftDistance,
          scale,
          boxShadow: addShadow
            ? `0 ${shadowBlur}px ${shadowBlur + shadowSpread}px rgba(0, 0, 0, ${shadowIntensity})`
            : undefined,
        },
        transition: {
          duration,
          ease,
        },
      };
    }
    ```

    **Step 2: Copy the component**

    ```tsx title="components/animations/buttons/lift.tsx"
"use client";
import { motion, type HTMLMotionProps } from "motion/react";
import { useHoverLift } from "@/lib/use-hover-lift";
interface LiftButtonProps extends Omit<HTMLMotionProps<"button">, "animate"> {
  children: React.ReactNode;
  liftHeight?: number;
  shadow?: boolean;
}
export function LiftButton({
  children,
  liftHeight = 8,
  shadow = true,
  ...props
}: LiftButtonProps) {
  const liftProps = useHoverLift({
    liftDistance: liftHeight,
    addShadow: shadow,
  });
  return (
    <motion.button {...liftProps} {...props}>
      {children}
    </motion.button>
  );
}
```

## Using the Hook Directly

For maximum flexibility, use the `useHoverLift` hook on any element:

```tsx
import { motion } from "motion/react"
import { useHoverLift } from "@/lib/hooks"

export function CustomElement() {
  const liftProps = useHoverLift({ liftDistance: 12 })
  
  return (
    <motion.div 
      {...liftProps} 
      className="inline-flex items-center gap-2 rounded-lg bg-white border border-neutral-200 p-6 shadow-sm"
    >
      <h3 className="font-semibold">Any Element</h3>
      <p className="text-sm text-neutral-600">Can have lift effect</p>
    </motion.div>
  )
}
```

## Combining with Other Effects

```tsx
import { motion } from "motion/react"
import { useHoverLift, useHoverGlow } from "@/lib/hooks"

export function LiftAndGlow() {
  const lift = useHoverLift({ liftDistance: 10 })
  const glow = useHoverGlow({ glowColor: "rgba(59, 130, 246, 0.5)" })
  
  const combined = {
    ...lift,
    whileHover: {
      ...lift.whileHover,
      ...glow.whileHover,
    },
  }
  
  return (
    <motion.button {...combined} className="inline-flex h-11 items-center gap-2 rounded-lg bg-blue-600 px-6 text-sm font-semibold text-white">
      Lift + Glow
    </motion.button>
  )
}
```

## Use cases

`LiftButton` delegates to `useHoverLift`, which animates `y` (translateY) and `boxShadow` on `whileHover`. The default `liftHeight` is 8 pixels (maps to `liftDistance` in the hook), the shadow blur is `liftDistance * 2` (16px), and `shadowIntensity` defaults to 0.3. The transition uses `duration: 0.2` with `easeOut`. Reach for it when:

- **Primary action buttons in card grids.** Cards that reveal an action button on hover benefit from the lift reinforcing the hover state without changing color or layout.
- **Navigation link buttons.** A pill-shaped nav item that lifts slightly on hover communicates interactivity more clearly than a color change alone.
- **Icon action buttons.** A bookmark, share, or favorite button using `liftHeight={6}` responds to hover with a tight, contained lift that doesn't disturb neighboring elements.
- **Secondary CTAs.** After a primary CTA, a "Learn more" or "See pricing" button can use `liftHeight={4}` for a softer lift that avoids competing with the primary.

The `shadow` prop defaults to `true`, so the lift always comes with a growing shadow that reinforces the vertical displacement. Set `shadow={false}` if your button already has a strong visual background and the shadow would be redundant.

## When to use / when not to use

**Use `LiftButton` when:**

- You want a hover effect that feels physical and grounded (the shadow makes the lift read as real depth).
- The button is on a surface with a solid or near-solid background so the shadow is visible.
- You need a hover interaction without any ongoing animation overhead (lift only fires on hover, not continuously).

**Reach for a different button when:**

- You want a colored glow rather than a neutral shadow on hover. Use [Glow Button](/docs/animations/buttons/glow), which animates `boxShadow` with a colored spread.
- The button is inside a flex row where a `translateY` shift would push adjacent elements. The lift moves only the button itself (not its layout box), so siblings don't shift, but the button may visually overlap them. Reduce `liftHeight` or use [Press Button](/docs/animations/buttons/press) instead for click-based feedback.
- You need both hover lift and click feedback. `LiftButton` only responds to `whileHover`. Combine with `whileTap={{ scale: 0.97 }}` passed as a prop, or use [Press Button](/docs/animations/buttons/press) alongside it.
- The button sits on a patterned or textured background where a neutral gray shadow won't read. Use a border or color change instead.

## Accessibility notes

`LiftButton` renders a `motion.button`, so it carries full native button semantics and participates in tab order.

- **The lift is purely visual.** Moving the button 8px upward via `translateY` does not reorder the DOM or change the element's accessible position. Screen readers are not affected.
- **Focus visibility.** The `whileHover` animation is mouse-triggered. Keyboard focus does not activate the lift, so ensure your CSS defines a `:focus-visible` ring (e.g., `focus-visible:ring-2 focus-visible:ring-blue-500`) to give keyboard users an equivalent indicator.
- **Don't use lift as a state signal.** If the lift is supposed to indicate "this button is selected" or "this action is dangerous," it won't be perceived by screen reader users. Use an `aria-pressed` or `aria-current` attribute and a text label.
- **Reduced motion.** The `y` transform and `boxShadow` changes are brief (0.2 seconds) and happen only during hover, not continuously. To eliminate the motion entirely:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { LiftButton } from "@/components/animations/buttons/lift"

function AccessibleLift({ children }: { children: React.ReactNode }) {
  const reduced = useReducedMotion()
  return (
    <LiftButton
      liftHeight={reduced ? 0 : 8}
      shadow={!reduced}
      className="inline-flex h-11 items-center rounded-lg bg-blue-600 px-6 text-sm font-semibold text-white focus-visible:ring-2 focus-visible:ring-blue-400"
    >
      {children}
    </LiftButton>
  )
}
```

## Performance considerations

- **`translateY` is compositor-safe.** The lift animates the `y` motion value, which maps to a CSS `transform: translateY(...)`. This runs on the compositor thread with no layout recalculation.
- **`boxShadow` triggers repaint.** When `shadow={true}` (the default), the shadow transitions on hover entry and exit, which requires a repaint each time. This is a single repaint per hover event, not a continuous one. It is negligible for one or a few buttons, but if you render many `LiftButton` components in a dense list and all are hovered simultaneously (e.g., in a card grid), consider `shadow={false}` and using a CSS `box-shadow` transition instead.
- **No ongoing animation.** Unlike [Border Gradient Button](/docs/animations/buttons/border-gradient), which loops indefinitely, `LiftButton` only schedules animation frames during the hover transition. When the user moves the mouse away, the animation completes and stops.
- **`useHoverLift` returns a static props object.** The hook computes shadow strings on each render but has no subscriptions or timers. Memoizing the parent component eliminates redundant calls.
