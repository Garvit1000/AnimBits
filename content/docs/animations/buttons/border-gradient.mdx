## Overview

The `BorderGradientButton` creates an animated gradient border that continuously traces around the button's perimeter, creating a premium, eye-catching effect.

"use client";
import * as React from "react";
import { motion, HTMLMotionProps } from "motion/react";
interface BorderGradientButtonProps
  extends Omit<HTMLMotionProps<"button">, "children"> {
  children: React.ReactNode;
  duration?: number;
  colors?: string[];
}
function cn(...classes: (string | undefined)[]) {
  return classes.filter(Boolean).join(" ");
}
export function BorderGradientButton({
  children,
  className,
  duration = 2,
  colors = ["#3b82f6", "#8b5cf6", "#ec4899"],
  ...props
}: BorderGradientButtonProps) {
  const buttonRef = React.useRef<HTMLButtonElement>(null);
  const [dimensions, setDimensions] = React.useState({
    width: 0,
    height: 0,
    radius: 0,
    perimeter: 0,
  });
  const gradientId = React.useId();
  React.useEffect(() => {
    if (!buttonRef.current) return;
    const updateDimensions = () => {
      if (buttonRef.current) {
        const styles = window.getComputedStyle(buttonRef.current);
        const width = buttonRef.current.offsetWidth;
        const height = buttonRef.current.offsetHeight;
        const borderRadius =
          parseFloat(styles.borderTopLeftRadius) ||
          parseFloat(styles.borderRadius) ||
          0;
        const maxRadius = Math.min(width, height) / 2;
        const actualRadius = Math.min(borderRadius, maxRadius);
        const straightEdges =
          2 * (width - 2 * actualRadius + (height - 2 * actualRadius));
        const curvedEdges = 2 * Math.PI * actualRadius;
        const perimeter = straightEdges + curvedEdges;
        setDimensions({
          width,
          height,
          radius: actualRadius,
          perimeter: perimeter || 1,
        });
      }
    };
    updateDimensions();
    const observer = new ResizeObserver(updateDimensions);
    if (buttonRef.current) {
      observer.observe(buttonRef.current);
    }
    return () => observer.disconnect();
  }, []);
  return (
    <motion.button
      ref={buttonRef}
      className={cn("relative", className)}
      {...props}
    >
      {}
      {dimensions.width > 0 && (
        <svg
          className="absolute inset-0 pointer-events-none"
          width={dimensions.width}
          height={dimensions.height}
        >
          <defs>
            <linearGradient id={gradientId} x1="0%" y1="0%" x2="100%" y2="100%">
              {colors.map((color, i) => (
                <stop
                  key={i}
                  offset={`${(i / (colors.length - 1)) * 100}%`}
                  stopColor={color}
                />
              ))}
            </linearGradient>
          </defs>
          <motion.rect
            x="1"
            y="1"
            width={dimensions.width - 2}
            height={dimensions.height - 2}
            rx={dimensions.radius}
            fill="none"
            stroke={`url(#${gradientId})`}
            strokeWidth="2"
            strokeDasharray={dimensions.perimeter}
            strokeDashoffset={0}
            animate={{ strokeDashoffset: [0, -dimensions.perimeter] }}
            transition={{
              duration,
              ease: "linear",
              repeat: Infinity,
            }}
            style={{
              strokeLinecap: "round",
            }}
          />
        </svg>
      )}
      <span className="relative z-10">{children}</span>
    </motion.button>
  );
}

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `duration` | `number` | `2` | Animation duration in seconds for one complete cycle |
| `colors` | `string[]` | `["#3b82f6", "#8b5cf6", "#ec4899"]` | Array of hex colors for the gradient |
| `children` | `React.ReactNode` | Required | Button content |
| `className` | `string` | `undefined` | Additional CSS classes |

All other props from `motion.button` are supported and passed through.

## Animation Details

The border gradient animation works by:

1. Using a rotatingparate animated divs for each side (top, right, bottom, left)
2. Each div moves along its respective edge with a sequential delay
3. The gradient flows smoothly as it traces around the perimeter
4. Animation runs continuously and infinitely
5. Each side's animation is perfectly timed to create a seamless loop

## Accessibility

- Maintains all standard button semantics
- Keyboard accessible (can be focused and activated)
- Animation respects `prefers-reduced-motion` when using Framer Motion's default settings
- Supports all ARIA attributes through prop spreading

## Browser Support

- Works in all modern browsers that support CSS gradients and transforms
- Requires Framer Motion for animations
- Gracefully degrades in browsers without animation support

## Use cases

The `BorderGradientButton` suits situations where you want to draw the eye to a button without changing its fill color. The animated `strokeDashoffset` traces the exact perimeter of the button, including its border radius, so the gradient feels painted on rather than overlaid. Reach for it when:

- **Hero CTAs.** A primary "Get started" or "Sign up free" button on a landing page benefits from the continuous motion, which naturally attracts peripheral vision.
- **Pricing card actions.** A highlighted plan's upgrade button needs to feel premium; the gradient border differentiates it from sibling buttons without requiring a different background.
- **Feature spotlight buttons.** Alongside a newly released feature callout, the animated border signals "pay attention here" without being as aggressive as a full background animation.
- **Dark-mode interfaces.** On dark backgrounds (`bg-neutral-900`), the default blue-purple-pink gradient (`colors` default) pops strongly while keeping the button content legible against the dark fill.

Because the SVG overlay measures actual DOM dimensions via `ResizeObserver` and reacts to `borderTopLeftRadius`, the animation adapts automatically to `rounded-lg`, `rounded-full`, and `rounded-none` without any extra config.

## When to use / when not to use

**Use `BorderGradientButton` when:**

- You need exactly one focal CTA per viewport that merits continuous animation.
- The button has a solid or semi-transparent background that contrasts with the gradient so the border is visible.
- The design calls for a premium feel on a contained element rather than a full-page effect.

**Reach for a different button when:**

- You have three or more primary buttons visible at once. Multiple spinning borders compete for attention and exhaust the eye; use a static [Glow Button](/docs/animations/buttons/glow) or [Lift Button](/docs/animations/buttons/lift) for secondary actions instead.
- The button sits on a light multi-color background where the default `["#3b82f6", "#8b5cf6", "#ec4899"]` gradient blends in. You will need to customize `colors` or pick a different component.
- The user needs to activate the button repeatedly in quick succession (forms, incrementers). A continuous looping border gives no click feedback; combine with [Press Button](/docs/animations/buttons/press) via `whileTap` if you need tap confirmation.
- The button is inside a data-dense table or list. Continuous motion in a tight grid is distracting.

## Accessibility notes

The `BorderGradientButton` is a `motion.button`, so it carries full button semantics by default. Still:

- **Keyboard and focus.** The component spreads all props onto `motion.button`, so `:focus-visible` styles from your CSS apply. Make sure your global styles or Tailwind config define a visible ring (e.g., `focus-visible:ring-2 focus-visible:ring-blue-500`) because the animated border is not a focus indicator.
- **Don't rely on the border color alone.** The gradient is purely decorative. Ensure the button label or an `aria-label` conveys the action; sighted users and screen reader users should both understand the purpose.
- **Reduced motion.** The border animation loops forever via `repeat: Infinity`. Honor the OS setting to stop the loop:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { BorderGradientButton } from "@/components/animations/buttons/border-gradient"

function AccessibleBorderGradient({ children }: { children: React.ReactNode }) {
  const reduced = useReducedMotion()
  // A very long duration makes the trace imperceptibly slow for users who prefer no motion.
  return (
    <BorderGradientButton
      duration={reduced ? 9999 : 2}
      className="inline-flex h-11 items-center rounded-lg bg-white dark:bg-neutral-900 px-6 text-sm font-semibold text-neutral-900 dark:text-white"
    >
      {children}
    </BorderGradientButton>
  )
}
```

- **ARIA attributes.** Pass `aria-disabled="true"` and `disabled` together when the button is inactive; the SVG overlay is `pointer-events-none` so it never intercepts clicks, but the `motion.button` itself must still be disabled to block keyboard activation.

## Performance considerations

- **Compositor-friendly animation.** The gradient traces by animating `strokeDashoffset` on an SVG `<rect>`. SVG stroke-dashoffset changes are handled by the compositor via an SVG paint layer; they do not trigger layout or pixel-level repaints of the surrounding DOM.
- **Infinite loop caveat.** `repeat: Infinity` keeps the animation scheduled indefinitely. Conditionally render the button only when it needs to be on screen; hiding it with `display:none` or `visibility:hidden` without unmounting still runs the animation. Unmounting stops it.
- **ResizeObserver per instance.** Each `BorderGradientButton` attaches one `ResizeObserver` to its button node for dimension tracking. This is lightweight for a handful of instances, but avoid rendering dozens of them in a list since each observer holds a separate callback slot.
- **`useId` for gradient IDs.** The component uses React's `useId` to generate unique `linearGradient` IDs, preventing SVG `defs` collisions when multiple instances share a page. No extra config needed.
- **SVG sizing.** The overlay SVG is sized via inline `width`/`height` attributes equal to the button's `offsetWidth` and `offsetHeight`. It does not stretch or reflow its container because it is `position: absolute` with `inset-0` and `pointer-events-none`.
