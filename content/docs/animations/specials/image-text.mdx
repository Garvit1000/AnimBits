## Image Text

A component that reveals an animated background image through text using `background-clip: text`.

"use client";

import React, { useState, useEffect } from "react";
import { motion, HTMLMotionProps, AnimatePresence } from "motion/react";
import { cn } from "@/lib/utils";

interface ImageTextProps extends HTMLMotionProps<"h1"> {
    text: string;
    imageUrl: string | string[] | { src: string } | { src: string }[];
    className?: string;
    direction?: "horizontal" | "vertical" | "diagonal" | "none";
    interval?: number;
}

export function ImageText({
    text,
    imageUrl,
    className,
    direction = "horizontal",
    interval = 3000,
    ...props
}: ImageTextProps) {
    const images = Array.isArray(imageUrl) ? imageUrl : [imageUrl];
    const [currentImageIndex, setCurrentImageIndex] = useState(0);

    useEffect(() => {
        if (!Array.isArray(imageUrl) || imageUrl.length <= 1) return;

        const timer = setInterval(() => {
            setCurrentImageIndex((prev) => (prev + 1) % images.length);
        }, interval);

        return () => clearInterval(timer);
    }, [imageUrl, interval, images.length]);

    const getCurrentImageUrl = () => {
        const img = images[currentImageIndex];
        return typeof img === "string" ? img : img.src;
    };

    const getAnimation = () => {
        switch (direction) {
            case "horizontal":
                return {
                    backgroundPosition: ["0% 50%", "100% 50%", "0% 50%"],
                };
            case "vertical":
                return {
                    backgroundPosition: ["50% 0%", "50% 100%", "50% 0%"],
                };
            case "diagonal":
                return {
                    backgroundPosition: ["0% 0%", "100% 100%", "0% 0%"],
                };
            case "none":
                return {
                    backgroundPosition: "center",
                }
            default:
                return {
                    backgroundPosition: ["0% 50%", "100% 50%", "0% 50%"],
                };
        }
    };

    return (
    <span className={cn("inline-block relative", className)}>
         {/* Invisible layout placeholder ensuring size doesn't collapse */}
        <h1 className={cn("text-9xl font-serif italic font-bold tracking-tighter inline-block invisible", className)}>
            {text}
        </h1>

        <AnimatePresence>
            <motion.h1
                key={currentImageIndex} // Re-render for image change
                className={cn(
                    "text-9xl font-serif italic font-bold tracking-tighter inline-block absolute inset-0",
                    className
                )}
                style={{
                    backgroundImage: `url(${getCurrentImageUrl()})`,
                    backgroundSize: direction === "none" ? "cover" : "150% auto",
                    backgroundClip: "text",
                    WebkitBackgroundClip: "text",
                    WebkitTextFillColor: "transparent",
                    color: "transparent",
                }}
                initial={{ opacity: 0 }}
                animate={{
                     ...getAnimation(),
                     opacity: 1
                }}
                exit={{ opacity: 0 }}
                transition={{
                    // Smooth opacity fade for texture change
                    opacity: { duration: 1 },
                    // Continuous background movement
                    backgroundPosition: {
                        duration: 15,
                        ease: "linear",
                        repeat: Infinity,
                    }
                }}
                {...props}
            >
                {text}
            </motion.h1>
        </AnimatePresence>
    </span>
);
}

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `text` | `string` | - | The text content to display. |
| `imageUrl` | `string \| string[] \| StaticImageData` | - | The URL(s) or imported image object(s) of the background. |
| `direction` | `"horizontal" \| "vertical" \| "diagonal" \| "none"` | `"horizontal"` | The direction of the background animation. |
| `interval` | `number` | `3000` | The duration (ms) to show each image when `imageUrl` is an array. |
| `className` | `string` | - | Additional classes for custom text styling (size, weight). |

## Use cases

`ImageText` uses `background-clip: text` with `WebkitBackgroundClip: text` and `WebkitTextFillColor: transparent` to reveal an animated image through the letterforms of the text. The background position animates continuously via a `motion/react` `backgroundPosition` keyframe with `repeat: Infinity` and `ease: "linear"` over 15 seconds. When `imageUrl` is an array, `AnimatePresence` fades between images on an `interval` timer.

- **Hero section wordmarks.** A single large word like a brand name or headline word filled with a landscape photo creates an immediate visual anchor for landing pages.
- **Category or section headers.** Fill section titles with thematic imagery (a forest photo for "Nature", a cityscape for "Urban") to tie typography to content.
- **Rotating seasonal imagery.** Pass an array of images with `direction="none"` and `interval={3000}` to cycle through seasonal or campaign photos as the text background.
- **Pull-quotes or statistics.** A bold number like "10x" or "500K" filled with a gradient photo makes a metric stand out on a marketing page.

## When to use / when not to use

**Use `ImageText` when:**

- The text is large (approximately `text-6xl` and above). Below that, `background-clip: text` produces an illegible smear rather than a readable image fill.
- You have one or two visually distinct words to highlight. The component renders a hidden `h1` placeholder for layout, then an absolutely-positioned animated `motion.h1` on top. Wrapping many lines of body text this way produces layout complications.
- The imagery is high-contrast relative to the page background so the text remains readable where the image is bright.

**Do not use `ImageText` when:**

- The text is body copy or small UI labels. Use a standard color or gradient text class instead.
- You need the text indexed by search engines as actual copy. `background-clip: text` renders real DOM text, so SEO is preserved, but the invisible layout placeholder `h1` is a duplicate element. Prefer wrapping with a semantic container and confirming your crawler sees the real content.
- The page background is similar in brightness to the image fill. The text will disappear. Ensure a contrasting matte background or use a `text-shadow` fallback.
- You are cycling images rapidly (`interval` below 1000 ms). The `AnimatePresence` fade uses a 1-second `opacity` transition. Images cycling faster than the fade duration will overlap visually.

## Accessibility notes

- **Semantic heading.** The component renders an `h1` element by default. If the text is not a page title, override with a `span` or `h2` by customizing the component to match the correct heading level in context. Do not place multiple `h1` elements on a page.
- **Color contrast.** `background-clip: text` with a photographic image does not have a predictable contrast ratio. Parts of the image that are close in luminance to the page background will make the text unreadable. Always verify contrast with your specific image, especially for users with low vision.
- **Fallback for unsupported environments.** Browsers that do not support `WebkitBackgroundClip: text` will render transparent text (invisible). Add a CSS fallback:

```css
/* In your global CSS */
@supports not (-webkit-background-clip: text) {
  .image-text-fallback {
    color: inherit;
    background: none;
  }
}
```

- **Reduced motion.** The `backgroundPosition` keyframe runs infinitely. Respect user preference:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { ImageText } from "@/components/animations/specials/image-text"

function AccessibleImageText(props: React.ComponentProps<typeof ImageText>) {
  const reduced = useReducedMotion()
  return <ImageText {...props} direction={reduced ? "none" : props.direction} />
}
```

- **Alt text.** Because the image is a CSS background, screen readers do not announce it. The visible `text` prop is the accessible label; no additional `alt` is required.

## Performance considerations

- **Two `h1` elements per instance.** The component renders a visually hidden (`invisible`) `h1` for layout and a positioned animated `motion.h1` on top. This is intentional to prevent layout collapse, but it means two heading elements in the DOM. In a page with many instances, this multiplies DOM nodes.
- **`backgroundPosition` with `repeat: Infinity`.** For `direction="horizontal"` (the default), the background animates from `0% 50%` to `100% 50%` and back over 15 seconds. This is a compositable CSS property on modern browsers but may fall back to main-thread painting on older engines.
- **`backgroundSize: "150% auto"`.** The image is scaled to 150% of the container width for all directions except `"none"`. Ensure the source image is large enough that the browser does not upscale a small image to fill the oversized background, which would look blurry.
- **`AnimatePresence` on image arrays.** Each image change mounts a new `motion.h1` and unmounts the previous one with a 1-second opacity fade. On slow connections the incoming image may not be loaded when the `interval` fires, causing a flash of the previous image. Preload images with `new Image().src = url` before passing them to the component.
- **Avoid many instances on a single page.** Each instance runs a continuous `requestAnimationFrame`-backed animation. Two or three instances are fine; ten on a scroll-driven page will consume compositor resources.
