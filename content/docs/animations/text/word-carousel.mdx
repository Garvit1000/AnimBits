    **Step 1: Copy the hook**

    ```tsx title="lib/hooks/use-word-carousel.ts"
    import * as React from "react"

    export interface UseWordCarouselOptions {
      words: string[]
      interval?: number
    }

    export function useWordCarousel(options: UseWordCarouselOptions) {
      const { words, interval = 2 } = options
      const [currentIndex, setCurrentIndex] = React.useState(0)

      React.useEffect(() => {
        const timer = setInterval(() => {
          setCurrentIndex((prev) => (prev + 1) % words.length)
        }, interval * 1000)

        return () => clearInterval(timer)
      }, [words.length, interval])

      return {
        currentWord: words[currentIndex],
        currentIndex,
        key: currentIndex,
      }
    }
    ```

"use client";
import * as React from "react";
import { motion, AnimatePresence, HTMLMotionProps } from "motion/react";
import { cn } from "@/lib/utils";
import {
  useWordCarousel,
  type UseWordCarouselOptions,
} from "@/lib/use-word-carousel";
export interface TextWordCarouselProps
  extends Omit<HTMLMotionProps<"span">, "children">,
    UseWordCarouselOptions {
  duration?: number;
}
export function TextWordCarousel({
  words,
  interval,
  className,
  duration = 0.3,
  ...props
}: TextWordCarouselProps) {
  const { currentWord, key } = useWordCarousel({ words, interval });
  return (
    <span className={cn("inline-block relative", className)}>
      <AnimatePresence mode="wait">
        <motion.span
          key={key}
          initial={{ opacity: 0, y: 20 }}
          animate={{ opacity: 1, y: 0 }}
          exit={{ opacity: 0, y: -20 }}
          transition={{ duration }}
          className="inline-block"
          {...props}
        >
          {currentWord}
        </motion.span>
      </AnimatePresence>
    </span>
  );
}

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `words` | `string[]` | Required | Array of words to cycle through |
| `interval` | `number` | `2` | Time between words (seconds) |
| `duration` | `number` | `0.3` | Transition duration (seconds) |

## Use cases

`TextWordCarousel` cycles words using the `useWordCarousel` hook, which runs `setInterval` at `interval * 1000` ms (default `interval=2` seconds). The hook returns `{ currentWord, currentIndex, key }`. The component wraps the current word in `<AnimatePresence mode="wait">` so the outgoing word exits (`opacity: 0, y: -20`) before the incoming word enters (`opacity: 0, y: 20` to `opacity: 1, y: 0`), each with a `duration` of 0.3 s. Reach for it when:

- **Hero tagline with multiple value propositions.** Inline within a heading: "Build [Fast / Modern / Beautiful] websites" cycles through adjectives without changing the surrounding copy.
- **Tech stack showcase.** "Built with [React / Next.js / TypeScript]" in a footer or about section.
- **Dynamic benefit statement.** "Save time on [deployments / reviews / testing]" cycles through pain points addressed.
- **Rotating testimonial attribution.** A name or company cycling below a quote.

Unlike [DiaText](/docs/animations/text/dia-text), `TextWordCarousel` uses a plain `y` slide without the multi-color gradient trace, making it a simpler, less visually assertive choice.

## When to use / when not to use

**Use word-carousel when:**

- The visual treatment should be subtle. The default 0.3 s slide with no color change is appropriate when the surrounding heading already has strong typography.
- You want consistent, reliable cycling with a clean API: pass `words`, optionally set `interval` and `duration`, done.
- The words in your array are similar in length. Width changes between words can shift surrounding text unless you apply a fixed width to the carousel span.

**Reach for something else when:**

- **You want a dramatic visual trace on each word change.** Use [DiaText](/docs/animations/text/dia-text), which adds a multi-color gradient sweep during entry.
- **You need the word change to pause after the last word.** Both `TextWordCarousel` and `DiaText` loop indefinitely. There is no `loop={false}` prop.
- **More than 4 to 5 words in the array.** Users typically see only 2 to 3 words before losing interest; longer lists cycle out-of-sight content. Keep the list short enough that users notice the pattern within the first two rotations.
- **The text must be screen-reader friendly per-word.** The cycling text is announced continuously as words change, which can be disruptive for screen reader users. See accessibility notes below.
- **You need a specific easing or spring on the transition.** The component hardcodes the exit/enter with `duration` and default easing. For a spring-based word change, use [DiaText](/docs/animations/text/dia-text) which uses `stiffness: 300, damping: 30`.

## Accessibility notes

The cycling animation is inside a plain `<span>`. Screen readers may announce each word as it rotates, which is disruptive mid-sentence. To prevent this, mark the cycling span as `aria-hidden` and provide a static `aria-label` on the parent element:

```tsx
<h1 aria-label="Build Fast websites">
  Build{" "}
  <span aria-hidden="true">
    <TextWordCarousel words={["Fast", "Modern", "Beautiful"]} className="text-blue-500" />
  </span>{" "}
  websites
</h1>
```

If the cycling content genuinely communicates different meaning to all users (not just visual flair), use `aria-live="polite"` on a companion visually-hidden element that mirrors the current word, and keep the visual carousel `aria-hidden`:

```tsx
"use client"
import { useState } from "react"
import { TextWordCarousel } from "@/components/animations/text/word-carousel"

// This pattern announces words on change without over-announcing.
// Maintain currentWord in state via the hook if exposing it is needed.
```

**Reduced motion.** The component animates `y` (translate) and `opacity` on each word change, indefinitely. Wrap to disable cycling under reduced motion:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { TextWordCarousel } from "@/components/animations/text/word-carousel"

function AccessibleCarousel({ words, ...props }: { words: string[] }) {
  const reduced = useReducedMotion()
  if (reduced) return <span {...props}>{words[0]}</span>
  return <TextWordCarousel words={words} {...props} />
}
```

## Performance considerations

- **`setInterval` runs indefinitely.** The interval is cleared on unmount via the effect cleanup. If the component mounts off-screen and stays mounted, it keeps cycling. Pair with an `IntersectionObserver` or a conditional mount if the carousel is below the fold for extended periods.
- **`AnimatePresence mode="wait"`.** The `wait` mode ensures the exit animation completes before the enter animation starts. At `duration=0.3`, each full transition (exit + enter) takes 0.6 s. During this window, both the exiting and entering `<motion.span>` are in the DOM briefly. At `interval=2 s`, this is well within tolerance.
- **No width stabilization by default.** The `<span>` wrapping the carousel does not fix its width to the widest word. If words differ in length, surrounding text will reflow on each change. Add `min-w-[Xch]` or `w-[Xch]` to the `className` to prevent layout shift:

```tsx
<TextWordCarousel
  words={["Fast", "Modern", "Beautiful"]}
  className="inline-block min-w-[9ch] text-center text-blue-500"
/>
```
