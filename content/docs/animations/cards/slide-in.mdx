    **Step 1: Copy the hook**
    
    ```tsx title="lib/hooks/use-slide-in.ts"
    import { type MotionProps, type Easing } from "motion/react";

    export type SlideDirection = "left" | "right" | "up" | "down";

    export interface UseSlideInOptions {
      direction?: SlideDirection;
      distance?: number;
      duration?: number;
      delay?: number;
      initialOpacity?: number;
      ease?: Easing | Easing[];
      viewport?: "once" | "always";
    }

    export function useSlideIn(options: UseSlideInOptions = {}): MotionProps {
      const {
        direction = "left",
        distance = 50,
        duration = 0.5,
        delay = 0,
        initialOpacity = 0,
        ease = "easeOut",
        viewport = "once",
      } = options;

      const getInitialPosition = () => {
        switch (direction) {
          case "left": return { x: -distance, y: 0 };
          case "right": return { x: distance, y: 0 };
          case "up": return { x: 0, y: -distance };
          case "down": return { x: 0, y: distance };
          default: return { x: -distance, y: 0 };
        }
      };

      const initialPosition = getInitialPosition();

      return {
        initial: {
          ...initialPosition,
          opacity: initialOpacity,
        },
        whileInView: {
          x: 0,
          y: 0,
          opacity: 1,
        },
        viewport: {
          once: viewport === "once",
        },
        transition: {
          duration,
          delay,
          ease,
        },
      };
    }
    ```

    **Step 2: Copy the component**
    
    ```tsx title="components/animations/cards/slide-in.tsx"
"use client";
import * as React from "react";
import { motion, HTMLMotionProps } from "motion/react";
import { cn } from "@/lib/utils";
import { useSlideIn, type SlideDirection } from "@/lib/use-slide-in";
export interface CardSlideInProps extends HTMLMotionProps<"div"> {
  duration?: number;
  delay?: number;
  direction?: SlideDirection;
  distance?: number;
}
export function CardSlideIn({
  children,
  className,
  duration = 0.5,
  delay = 0,
  direction = "left",
  distance = 50,
  ...props
}: CardSlideInProps) {
  const slideProps = useSlideIn({
    direction,
    distance,
    duration,
    delay,
  });
  return (
    <motion.div {...slideProps} className={cn(className)} {...props}>
      {children}
    </motion.div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `duration` | `number` | `0.5` | Animation duration in seconds |
| `delay` | `number` | `0` | Delay before animation starts |
| `direction` | `"left" \| "right" \| "up" \| "down"` | `"left"` | Slide direction |
| `distance` | `number` | `50` | Distance to slide in pixels |
| `className` | `string` | `undefined` | Additional CSS classes |

## Use cases

`CardSlideIn` uses `whileInView` (once by default) to transition from `{ x: -50, opacity: 0 }` (for `direction="left"`) to `{ x: 0, opacity: 1 }`. The `useSlideIn` hook maps each direction to an initial `x` or `y` offset of `+/-distance` (default `50px`) and animates to zero on both axes. The `easeOut` easing makes the slide decelerate naturally at the end.

Use it when:

- **Feature sections with a clear reading direction.** Slide cards in from the left as the user scrolls down a left-to-right reading layout.
- **Two-column split layouts.** Left column slides from left, right column from right, meeting in the middle.
- **Notification or feed items.** New items that push into view from the top or bottom.
- **Step-by-step process cards.** Each numbered step slides in from below with a small `delay` increment.
- **Dashboard panels loaded after data fetch.** Slide in the panel from below once the data arrives.

## When to use / when not to use

**Use `CardSlideIn` when:**

- You want a viewport-triggered entrance with a clear spatial direction that reinforces the page layout (e.g., left-side content slides in from the left).
- You want combined translation + opacity, which `useSlideIn` provides together.

**Reach for a different component when:**

- You want a scale entrance rather than a slide. Use [CardScaleIn](/docs/animations/cards/scale-in) for a zoom-in effect.
- The card is above the fold at page load. `whileInView` fires immediately for visible elements, but [CardFadeIn](/docs/animations/cards/fade-in) is semantically cleaner for always-visible content.
- `distance={50}` causes the card to appear from outside its parent container. If the parent has `overflow: hidden`, the initial off-screen position will be invisible anyway (which is fine), but the clipping can make the entrance feel abrupt. Reduce `distance` to `20` or `30` for contained layouts.
- The slide direction contradicts the reading flow. Sliding cards in from the right in a left-to-right document feels counterintuitive. Match direction to the natural scroll or reading axis.

## Accessibility notes

- **Starting position is `opacity: 0`** and offset by `distance` pixels. Screen readers access the content immediately from the DOM regardless of visual state. Do not hide the card from the accessibility tree during the animation.
- **Translation is motion.** A 50px slide, while moderate, is a positional animation. Honor `prefers-reduced-motion` by skipping both the translate and the opacity fade:

```tsx
"use client"
import { useReducedMotion } from "motion/react"
import { CardSlideIn } from "@/components/animations/cards/slide-in"

function AccessibleSlideCard({ children }: { children: React.ReactNode }) {
  const reduced = useReducedMotion()
  return (
    <CardSlideIn
      distance={reduced ? 0 : 50}
      duration={reduced ? 0 : 0.5}
      direction="left"
      className="rounded-xl border border-neutral-200 dark:border-neutral-800 bg-white dark:bg-neutral-900 p-6"
    >
      {children}
    </CardSlideIn>
  )
}
```

- **`viewport: { once: true }` is the default.** This means the card slides in once and stays put. If you set `viewport="always"`, the card will re-animate every time it leaves and re-enters the viewport, which can be disorienting. Leave it at `"once"` for most use cases.
- **Keyboard navigation.** The card is in the normal tab order from the start. Keyboard users can reach a `CardSlideIn`-wrapped card while it is still animating. Ensure focus rings are visible throughout the animation.

## Performance considerations

- **`x`/`y` translation and `opacity` are GPU-composited.** The animation runs off the main thread and does not trigger layout or paint.
- **`IntersectionObserver` via `whileInView`.** One observer per card; cheap. After the animation completes with `once: true`, the observer disconnects.
- **`distance={50}` can cause a brief layout gap.** The initial offset positions the card 50px away from its final position. In some layouts, particularly flex or grid containers without fixed heights, this can shift surrounding elements during the entrance. Fix by adding a fixed height to the card container, or use `overflow: hidden` on the row.
- **Avoid large `distance` values.** At `distance={200}`, the card starts far enough off that the slide takes a noticeable path. Values above `80` should be paired with a longer `duration` (0.7 or more) to avoid a too-fast whip-in.

### Staggered step cards

```tsx
const steps = ["Research", "Design", "Build", "Launch"]

export function ProcessSteps() {
  return (
    <div className="flex flex-col gap-4">
      {steps.map((step, i) => (
        <CardSlideIn
          key={step}
          direction="left"
          distance={40}
          delay={i * 0.1}
          duration={0.4}
          className="rounded-xl border border-neutral-200 dark:border-neutral-800 bg-white dark:bg-neutral-900 p-5 flex items-center gap-4"
        >
          <span className="text-2xl font-black text-neutral-200 dark:text-neutral-700 w-8 shrink-0">{i + 1}</span>
          <p className="font-semibold">{step}</p>
        </CardSlideIn>
      ))}
    </div>
  )
}
```

```tsx
"use client"
import { useState } from "react"
import { CardSlideIn } from "@/components/animations/cards/slide-in"

interface Notification {

export function NotificationFeed({ items }: { items: Notification[] }) {

          <p className="text-sm">{n.message}</p>
          <span className="text-xs text-neutral-400 shrink-0 ml-3">{n.time}</span>

      ))}

}
```
